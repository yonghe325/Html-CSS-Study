DTD全称"**Document Type Definition**", 继承自SGML语言
==定义了 XML/HTML 文档结构规则的**语法规范文件**==
规定了文档的语法, 例如文档中可以出现那些标签, 标签可以有那些属性

他的核心功能是**验证**
解析器可以将文档与DTD比较, 检查代码是否符合规范
# DTD的位置 
DTD并不存在于文档中, 而是一份独立的,外部的`.dtd`文件
而这份文件可以通过 URL 引用
+ 内部DTD : 直接把规则写在 `<!DOCTYPE>` 声明的方括号 `[]` 里
	转到[[1. DocumentType 的变化#`internalSubset`|internalSubset]]
+ 外部DTD, 通过[[1. DocumentType 的变化#`publicId`、`systemId`|systemId]]指向外部DTD
# HTML5中的DTD
==HTML5中没有DTD==
在HTML5中, 直接使用文档声明`<!DOCTYPE html>`告诉解析器以何种模式渲染文档
	由标准模式, 混杂模式等
所以现代浏览器不再依赖外部 DTD 文件来解析 HTML
它们直接按照 W3C 发布的 HTML5 标准规范来实现解析算法
因此通过`internalSubset`属性自定义规则也不能在HTML5中使用
