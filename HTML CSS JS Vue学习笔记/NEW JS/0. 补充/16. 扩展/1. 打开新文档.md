# 序列化文档
使用`window.open()`方法可以打开一个URL
所以, 就需要将新文档转化为一个URL
可以将新文档序列化为一个字符串, 然后生成一个临时的URL, 使用`window.open()`方法将其打开
1. 创建新文档
```js
const newDoc = document.implementation.createHTMLDocument("我的页面");
newDoc.body.innerHTML = '<h1>Hello, 新文档!</h1><p>这是一个独立的新页面。</p>';
```
2. 将新文档序列化为字符串
```js
const serializer = new XMLSerializer();
const docString = serializer.serializeToString(newDoc);
// serializeToString()方法, 返回newDoc完整的DOM树结构
```
3. 从字符串创建一个 Blob (可以理解为数据“块”)
```js
//    设置MIME类型为 text/html，告诉浏览器这是一个HTML文件
const blob = new Blob([docString], { type: 'text/html' });
```
4. 为这个 Blob 创建一个临时的 URL
```js
const blobUrl = URL.createObjectURL(blob);
```
5. 使用 `window.open` 打开这个 URL
```js
//在一个新标签页中打开你的内容
const newWindow = window.open(blobUrl);
```
6. 当不再需要时，释放这个临时 URL 以节省内存 (可选，但推荐)
```js
// setTimeout(() => URL.revokeObjectURL(blobUrl), 1000);
```
## XMLSerializer对象
XMLSerializer 是将 文档ODM树 序列化为字符串的API
可以把JavaScript代码中创建/修改的新文档, 变成一个可以保存, 传输或储存的==文本==

新创建的文档存储在内存中, 使用XMLSerializer API就可以将其固定保存 
### serializeToString() 方法
接受一个文档, 返回该文档的完整的DOM树结构
与innerHTML类似, 但innerHTML是能返回部分文档结构
例如 documentType对象等不会返回
