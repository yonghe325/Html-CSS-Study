**消息队列**是 JavaScript 事件循环机制中的一个重要组成部分
用于临时存放待执行的异步任务回调，等待*主线程*空闲时取出执行
具体过程如下:
``` deepseek
┌─────────────────────────────────────────────────────────┐
│                      主线程                              │
│  ┌─────────────────────────────────────────────────┐    │
│  │  执行同步代码 → 遇到异步 → 交给 Web API         │    │
│  │       ↓                                         │    │
│  │  继续执行后续同步代码                            │    │
│  │       ↓                                         │    │
│  │  同步代码执行完毕，检查消息队列                   │    │
│  └─────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────┘
                              ↑
                              │ 取出任务
┌─────────────────────────────┼───────────────────────────┐
│                     消息队列 │                           │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐                │
│  │ 任务1    │ │ 任务2    │ │ 任务3    │                │
│  │ (回调函数)│ │ (回调函数)│ │ (回调函数)│                │
│  └──────────┘ └──────────┘ └──────────┘                │
│         ↑           ↑           ↑                       │
│         └───────────┴───────────┘                       │
│              异步完成后放入                              │
└─────────────────────────────────────────────────────────┘
```
# 队列类型
队列有两个类型
**宏任务类型** 与 **微任务类型**
当从队列里取出任务时, 优先取出微任务队列里的任务
只有当微任务队列为空时, 才会从宏任务队列中取出任务
所以程序会严格按照以下顺序执行:
	同步代码->微任务->宏任务
无论setTimeout设定的时间是否已到,都必须按照该顺序执行
同一个任务类型队列中, 严格按照入队顺序执行
## 微任务
- 在当前宏任务执行完**立即执行**
- 用于需要==**尽快执行**==的操作（如 Promise 回调）
- 会阻塞渲染（因为渲染在宏任务之后）
	关于渲染, 转到[[渲染]]
## 宏任务
- 在微任务队列清空后执行
- 用于==**不紧急**==的操作
- 多个宏任务之间会进行 UI 渲染
---
任务类型由调用它的 API 内部实现决定
每个异步 API 在设计时就规定了它的回调是宏任务还是微任务
但是`async`与`await`会改变任务类型
