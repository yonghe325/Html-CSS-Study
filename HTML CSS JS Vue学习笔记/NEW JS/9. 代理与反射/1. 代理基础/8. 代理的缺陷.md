>代理是在 ECMAScript 现有基础之上构建起来的一套新 API
>因此其实现已经尽力做到最好了
>很大程度上，代理作为对象的虚拟层可以正常使用
>但在某些情况下
>代理也不能与现在的 ECMAScript机制很好地协同
# 代理中的`this`问题
众所周知`this`指向调用这个方法的对象
因此 使用代理调用被代理对象中的方法 与 通过对象直接调用其方法
方法里的`this`指向不同
```js
const target = {
	thisValEqualsProxy() {
		return this === proxy;
	}
}

const proxy = new Proxy(target, {});
console.log(target.thisValEqualsProxy()); // false
	//此时this为target
console.log(proxy.thisValEqualsProxy()); // true
	//此时this为代理对象proxy
```
这是符合直觉的
但是如果 目标对象 直接依赖于 对象标识
则会出现问题
例如在`WeakMap`中, 键为一个对象, 值通过键索引
实现类似于私有变量的效果
```js
const wm = new WeakMap();
class User {
	constructor(userId) {
		wm.set(this, userId);
	}
	set id(userId) {
		wm.set(this, userId);
	}
	get id() {
		return wm.get(this);
	}
}
```
由于`wm`中的值依赖`User`实例的对象标识
如果这个实例被代理, 就会有以下问题
```js
const user = new User(123);
console.log(user.id); // 123
const userInstanceProxy = new Proxy(user, {});
	//被代理
console.log(userInstanceProxy.id); // undefined
	//无法索引 , 因为此时this为代理对象
```
要解决这个问题
就需要重新配置代理
就需要把代 理 `User` 实例 改为 ==代理 `User` 类本身==
然后再通过类的代理实例化一个对象
```js
const UserClassProxy = new Proxy(User, {});
	//创建代理
const proxyUser = new UserClassProxy(456);
	//通过类的代理实例化对象
console.log(proxyUser.id);
	//此时wm中的键就为
```

```js
const UserClassProxy = new Proxy(User, {});
const user = new UserClassProxy(456);
const proxyuser = new Proxy(user,{});
console.log(user.id);
console.log(proxyuseruser.id);

```
# 代理 与 内部槽位
转到[[内部槽位|内部槽位]]
>代理与内置引用类型（比如 `Array`）的实例通常可以很好地协同
>但有些 ECMAScript 内置类型可能会依赖代理无法控制的机制
>结果导致在代理上调用某些方法会出错
>==因为代理对象可能没有被代理对象的某些内部槽位==

>一个典型的例子就是 `Date` 类型
>根据 ECMAScript 规范，`Date` 类型方法的执行依赖 `this` 值上的内部槽位`[[NumberDate]]`
>而 代理对象 没有内部槽位`[[NumberDate]]`
>	因为引擎没有为代理对象分配该内部槽位
>而且这个内部槽位的值也不能通过普通的 `get()`和 `set()`操作访问到
>	因为内部槽位只能由引擎操作 
>于是代理拦截后本应转发给目标对象的方法会抛出 异常：
```js
const target = new Date();
const proxy = new Proxy(target, {});
console.log(proxy instanceof Date); // true
proxy.getDate(); // TypeError: 'this' is not a Date object
```
## 解决方式
+ 不要代理某些拥有特定内部槽位的类型
	例如`Map`,`Set`,`Date`等
+ 直接绑定(bind)方法到目标对象
	关于`bind`,转到[[bind()|bind()方法]]
```js
const date = new Date();
const proxy = new Proxy(date, {
    get(target, prop, receiver) {
        const value = Reflect.get(target, prop, receiver);
	        //此时target.prop完全等价于 date.getDate
        if (typeof value === 'function') {
				// 如果value是一个方法 
				// 将该方法内的this值绑定到被代理对象上
				// 而不是默认的代理对象receiver上
            // 绑定到 target，而不是 proxy
            return value.bind(target);
	        //返回一个新函数, 它内部的this值始终是target
        }
        return value;
        //
    }
});

console.log(proxy.getDate());  // 正常工作
//proxy.getDate就是新函数, proxy.getDate()即调用该函数
//即调用date.getDate方法
```
+ 代理类, 而不是直接代理对象
```js
// 代理类，而不是代理实例
const DateProxy = new Proxy(Date, {
    construct(target, args) {
        const instance = new target(...args);
        // 返回普通实例，不返回代理
        return instance;
    }
});

const date = new DateProxy();
console.log(date.getDate());  // 正常（因为 date 是普通 Date 实例）
```
+ 使用普通对象模拟内置对象, 然后代理该对象
```js
// 使用普通对象存储数据，不使用 Date
const myDate = {
    timestamp: Date.now(),
    getDate() {
        return new Date(this.timestamp).getDate();
    }
};

const proxy = new Proxy(myDate, {});
console.log(proxy.getDate());  // 正常（没有内部槽位问题）
```
