虽然可以在捕获内部返回被捕获的原操作
但有些时候, 应该优先使用反射API
# 反射API 与 对象API
1. 反射API可以在捕获外部使用
2. 大多数反射API都有对应的`Object`静态方法
	反射API 与 对应的`Object`静态方法, 转到[[0. Proxy 捕获器完整参考表]]
	通常，Object 上的方法适用于通用程序
	而反射方法适用于细粒度的对象控制与操作
## 外部使用反射API
使用反射API可以代替很多只有操作符才能实现的操作
+ `Reflect.get()`：可以替代对象属性访问操作符。
+ `Reflect.set()`：可以替代`=`赋值操作符。
+ `Reflect.has()`：可以替代 `in` 操作符或 `with()`。
+ `Reflect.deleteProperty()`：可以替代 `delete` 操作符。
+ `Reflect.construct()`：可以替代 `new` 操作符。
# 状态标记
在使用某些反射方法时, 反射方法会返回一个名叫"状态标记"的`bool`值
表示意图执行的操作是否成功执行
例如 :
```js
// 初始代码
const o = {};
try {
	Object.defineProperty(o, 'foo', 'bar');
	console.log('success');
} catch(e) {
	console.log('failure');
}
```
如果使用`Object.defineProperty()`方法为对象`o`添加`foo`,`bar`属性不成功
该方法会直接抛出异常并被`try catch`捕获
但如果使用对应的反射方法. 不会抛出异常, 而是会返回一个`bool`值
	即为状态标记
此时就可以不用`try catch`, 省去系统开销
```js
// 重构后的代码
const o = {};
if(Reflect.defineProperty(o, 'foo', {value: 'bar'})) {
	//成功则为true
	console.log('success');
} else {
	//不成功则返回false
	console.log('failure');
}
```
+ 以下方法都提供状态标记
	 + `Reflect.defineProperty()`
	+ `Reflect.preventExtensions()`
	+ `Reflect.setPrototypeOf()`
	+ `Reflect.set()`
	+ `Reflect.deleteProperty()`

# 安全地使用函数
在通过 `apply()` 方法调用函数时
被调用的函数可能也定义了自己的 `apply()` 方法
	(虽然可能性很小)
此时原生的`apply()`方法被函数自定义的`apply()`方法遮盖
```js
function myFunction() {
    console.log('original');
}
// 给函数添加自定义的 apply 属性
myFunction.apply = function() {
    console.log('custom apply');
};
// 正常情况下调用 apply
myFunction.apply(null, []);
// 输出: "custom apply"（不是 "original"！）
// 因为函数自己的 apply 属性遮蔽了原生方法
```
而使用对应的`Reflect.apply()`反射API, 就可以避免这种情况`
```js
Reflect.apply(myFunction,null,[])
```
