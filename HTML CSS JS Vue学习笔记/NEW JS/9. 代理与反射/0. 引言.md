代理和反射提供了拦截并向基本操作嵌入额外行为的能力
+ 代理(Proxy) 
	拦截操作 和 自定义对象的 基本操作
+ 反射(Reflect)
	提供默认行为的调用方式
---
我来以我的理解解释一下, 你来判断我说的对不对:
1. 代理, 类似于c++中的引用, 修改/使用代理, 被代理的对象也会被修改/使用
	+ 会
2. 在使用被代理的对象时, 使用该对象的方法/属性, 会先调用代理中为该属性/方法的"前置方法"
	然后再使用该对象的方法/属性
	+ 对, "使用"这个行为会被代理捕获
		准确来说实惠被代理内的捕获器捕获, 然后执行相应的代码
3. 反射就是在代理执行完该方法/属性"前置方法"后, 再执行被代理对象的方法/属性
	+ 对, 使用反射方法可以为多个属性/方法设置反射
4. 被代理的对象还能使用吗, 如果直接使用被代理的对象, 还会执行反射吗?
	+ 可以使用, 直接使用不会被捕获
5. 代理对象被撤销后, 会被直接销毁吗?
	+ 不会被销毁, 会在内部标记为已销毁
		之后任何使用该代理的行为都会抛出异常
