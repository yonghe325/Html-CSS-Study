当使用数组字面量创建数组时
可以使用一串逗号`,`来创建空位(hole)
ECMAScript 会将逗号之间相应索引位置的值当成空位
```js
//ES6之前
const options = [,,,,,]; // 创建包含 5 个元素的数组
console.log(options.length); // 5
console.log(options); // [,,,,,]
//空位不是undefined

---------------------------------------------

//ES6之后
const options = [1,,,,5];
for (const option of options) {
console.log(option === undefined);
}
// false
// true
// true
// true
// false
//空位是undefined
```
数组空位（empty）与 `undefined` 不同：
- 空位不会被 `map()`、`forEach()` 等方法遍历
- 空位在 `in` 操作符中返回 `false`
- 空位在控制台中显示为 `empty` 或空

由于ES6前后多数组空位的解释不同,这里不再论述数组空位
==不要使用数组空位==
可以显式的将空位设置为`Undefined`
