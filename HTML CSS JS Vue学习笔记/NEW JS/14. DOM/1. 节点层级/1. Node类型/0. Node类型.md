DOM Level 1 描述了名为 Node 的接口，==这个接口是所有 DOM 节点类型都必须实现的==
Node 接口在 JavaScript中被实现为 Node 类型
在除 IE之外的所有浏览器中都可以直接访问这个类型
在 JavaScript中，==所有节点类型都继承 Node 类型==
	即Node类型是所有节点类型的父类
因此所有类型都共享相同的基本属性和方法
每个节点都有 `nodeType` 属性，表示该节点的类型
节点类型由定义在 Node 类型上的 12 个==数值常量==表示：(有用的节点已经标注)
+ `Node.ELEMENT_NODE`（1）: 元素节点
	转到[[0. Element类型|Element类型]]
+ Node.ATTRIBUTE_NODE（2）: 属性节点(==已废弃==)
	转到[[属性节点  Attr 对象]]
+ `Node.TEXT_NODE`（3）: 文本节点
	转到[[0. Text类型|Text类型]]
+ Node.CDATA_SECTION_NODE（4）: CDATA节点
	转到[[0. CDATASection类型|CDATASection类型]]
+ Node.ENTITY_REFERENCE_NODE（5）: 实体引用节点(==已废弃==)
+ Node.ENTITY_NODE（6）: 实体定义节点(==已废弃==)
+ Node.PROCESSING_INSTRUCTION_NODE（7）: 指令节点
+ `Node.COMMENT_NODE`（8）: 注释节点
	转到[[0. Comment类型|Comment类型]]
+ `Node.DOCUMENT_NODE`（9）: 文档根节点
	转到[[0. Document类型|Document类型]]
+ `Node.DOCUMENT_TYPE_NODE`（10）: 文档类型声明结点(`<!DOCTYPE>`元素)
	转到[[0. DocumentType类型|DocumentType类型]]
+ `Node.DOCUMENT_FRAGMENT_NODE`（11）: 文档片段节点
	转到[[0. DocumentFragment类型|DocumentFragment类型]]
+ Node.NOTATION_NODE（12）: DTD符号节点(==未实现,已废弃==)

---
节点类型可通过与这些常量比较来确定
```js
if (someNode.nodeType == Node.ELEMENT_NODE){
	alert("Node is an element.");
}
```
这个例子比较了 `someNode.nodeType` 与 `Node.ELEMENT_NODE` 常量
如果两者相等，则意味着 someNode 是一个元素节点
浏览器并不支持所有节点类型
开发者最常用到的是 元素节点(`Node.ELEMENT_NODE` 和 文本节点(`Node.TEXT_NODE`)
本章后面会讨论每种节点受支持的程度及其用法
