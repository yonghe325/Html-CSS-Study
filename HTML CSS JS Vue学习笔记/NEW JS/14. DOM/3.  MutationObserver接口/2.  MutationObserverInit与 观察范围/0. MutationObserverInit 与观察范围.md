MutationObserverInit 对象用于控制对目标节点的观察范围
粗略地讲，观察者可以观察的事件包括属性变化、文本变化和子节点变化
# MutationObserverInit对象的属性
+ `subtree` : 布尔值, 转到[[4. 观察子树#`subtree`属性->观察子树|subtree属性]]
	==表示除了目标节点，是否观察目标节点的子树（后代）==
	如果是 false，则只观察目标节点的变化
	如果是 true，则观察目标节点及其整个子树
	默认为 false
+ `attributes` : 布尔值
	==表示是否观察目标节点的属性变化==
	默认为 false
+ `attributeFilter` : 字符串数组, 转到[[1. 观察属性#attributeFilter属性 ->添加白名单|attributeFilter属性]]
	==表示要观察哪些属性的变化==
	把这个值设置为 true 也会将 attributes 的值转换为 true
	默认为观察所有属性
+ `attributeOldValue` : 布尔值, 转到[[1. 观察属性#`attributeOldValue`属性->保存旧值|attributeOldValue属性]]
	==表示 MutationRecord 是否记录变化之前的属性值==
	把这个值设置为 true 也会将 attributes 的值转换为 true
	默认为 false
+ `characterData` : 布尔值, 转到[[2. 观察字符数据#`characterData`属性->观察字符|characterData属性]]
	==表示修改字符数据是否触发变化事件==
	默认为 false
+ `characterDataOldValue` : 布尔值, 转到[[2. 观察字符数据#`characterDataOldValue`属性 -> 记录旧值|characterDataOldValue属性]]
	==表示 MutationRecord 是否记录变化之前的字符数据==
	把这个值设置为 true 也会将 characterData 的值转换为 true
	默认为 false
+ `childList` : 布尔值, 转到[[3. 观察子节点#`childList`属性-> 观察子代|childList属性]]
	==表示修改目标节点的子节点是否触发变化事件==
	默认为 false

---
# 注意
>在调用 `observe()`时
>`MutationObserverInit` 对象中的 attribute、characterData 和 childList 属性
>==必须至少有一项为 true==
>	（无论是直接设置这几个属性 ,还是通过设置 attributeOldValue 等属性
>		间接导致它们的值转换为 true）
>否则会抛出错误，==因为没有任何变化事件可能触发回调==
