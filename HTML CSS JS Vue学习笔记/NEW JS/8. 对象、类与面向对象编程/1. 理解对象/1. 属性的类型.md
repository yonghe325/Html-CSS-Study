属性的特征有实现JavaScript的引擎定义
开发者不能在 JavaScript 中直接访问这些特性
为了将某个特性标识为内部特性
	即属性级别的内部槽位
	转到[[内部槽位#属性级别的内部槽位|属性级别的内部槽位]]
规范会用两个中括号把特性的名称括起来，比如\[\[\Enumerable]]
属性分为两种 : ==数据属性==和==访问器属性==
# 数据属性
数据属性用来存储对象的数据
数据属性包含一个保存数据值的位置
值会从这个位置读取，也会写入到这个位置
+ 数据属性有 4个特性描述它们的行为 : 
	+ `[[Configurable]]` : 
		表示属性是否能通过`delete`删除并重新定义
		是否可以修改它的特性
		是否可以把这个属性改为访问器属性
		所有==直接定义==在对象上的属性的这个特性都是 true
	+ `[[Enumerable]]` : 
		表示该属性是否能通过`for-in`循环返回
		所有==直接定义==在对象上的属性的这个特性都是 true
	+ `[[Writable]]` : 决定是否==只读==
		表示属性的值是否可以被修改
		所有==直接定义==在对象上的属性的这个特性都是 true
	+ `[[Value]]` : 
		包含属性实际的值
		值会从这个位置读取，也会写入到这个位置
		这个特性的默认值为`Undefined`
# `Object.defineProperty()`修改特性
当一个属性被显式的添加时
`[[Configurable]]`,`[[Enumerable]]`,`[[Writable]]`会默认设为`true`,`Value`这会被修改为指定的值
使用`Object.defineProperty()`函数可以在添加数据属性时, 将四个特性进行修改
```js
Object.defineProperty(<对象>,<属性键>,<{
	[configurable: false/true,]
	[Enumerable: false/true,]
	[Writable: false/true,]
	//不指定, 默认均为false
	<value : <属性值>>
}>)
let person = {};
Object.defineProperty(person, "name", {
	configurable: false,
	value: "Nicholas"
});
console.log(person.name); // "Nicholas"
delete person.name;
console.log(person.name); // "Nicholas"

```
当第一次将一个数据属性的`[[configurable]]`修改为`false`后
再使用`Object.defineProperty()`为该属性修改将会受到限制
# 访问器属性
访问器属性不包含值
所以访问器属性没有`[[Writable]]`与`[[value]]`特性
只有`[[Configurable]]`与`[[Enumerable]]`特性
但包含非必须的一个获取`getter` 函数, 即`[[Get]]`特性 
与非必须的一个设置`setter`函数, 即`[[Set]]`特性
+ `[[Get]]` : 获取函数, 在读取属性时调用, 默认值为`Undefined`
	+ 只定义`get`函数, 代表该属性为==只读==
		非严格模式下试图读取该属性的值, 返回`Undefined`
		严格模式下视图修改该属性, 会抛出异常
+ `[[Set]]` : 设置函数, 在写入属性时调用, 默认值为`Undefined`
	+ 自定义`set`函数, 代表该属性为==只写==
		非严格模式下试图读取该属性的值, 返回`Undefined`
		严格模式下试图读取该属性的值, 会抛出异常
要定义访问器属性, 只能通过`Object.defineProperty()`函数
```js
// 定义一个对象，包含伪私有成员 year_和公共成员 edition
let book = {
year_: 2017,
edition: 1
};
Object.defineProperty(book, "year", {
	get() {
		return this.year_;
	},
	set(newValue) {
		if (newValue > 2017) {
			this.year_ = newValue;
			this.edition += newValue - 2017;
		}
	}
});
book.year = 2018;
//此时调用set函数
console.log(book.edition); // 2
```
