合并(Merge)
把源对象的所有的*本地属性*一起复制到目标对象上
也被称为 混入(mixin)
# `Object.assign()`方法
```js
Object.assign(<目标对象>,<源对象>[,<源对象1>,<源对象2>,<源对象3>])
```
+ 作用 : 
	将每个源对象中: 
		可枚举（`Object.propertyIsEnumerable()`返回 true）
			关于枚举的顺序, 转到[[6. 属性枚举顺序|属性枚举顺序]]
		自有(实例属性)（`Object.hasOwnProperty()`返回 true）的
		属性直接复制到`目标对象`(==不创建`目标对象`的副本==)
+ 参数 : 
	接收一个`目标对象`, 接收多个`<源对象>`
+ 返回值:
	返回被修改后的`目标对象`

```js
let dest, src, result;
/**  简单复制 **/
dest = {};
src = { id: 'src' };
result = Object.assign(dest, src);
// Object.assign 修改目标对象
// 也会返回修改后的目标对象
console.log(dest === result); // true
console.log(dest !== src); // true
console.log(result); // { id: src }
console.log(dest); // { id: src }

/** 多个源对象 **/
dest = {};
result = Object.assign(dest, { a: 'foo' }, { b: 'bar' });
console.log(result); // { a: foo, b: bar }
```
## 特点
### 浅复制 + 覆盖	
`Object.assign()`实际上对每个源对象执行的是==浅复制==
如果多个源对象都有相同的属性，则使用最后一个复制的值(覆盖)
```js
/** 属性覆盖 **/
dest = { id: 'dest' };
result = Object.assign(dest, { id: 'src1', a: 'foo' }, { id: 'src2', b: 'bar' });
// Object.assign 会覆盖重复的属性
console.log(result); // { id: src2, a: foo, b: bar }
// 可以通过目标对象上的设置函数观察到覆盖的过程：
dest = {
	set id(x) {
		console.log(x);
	}
};
Object.assign(dest, { id: 'first' }, { id: 'second' }, { id: 'third' });
// first
// second
// third

/** 浅复制 **/
dest = {};
src = { a: {} };
Object.assign(dest, src);
// 浅复制意味着只会复制对象的引用
console.log(dest); // { a :{} }
console.log(dest.a === src.a); // true
```

### 无法转移 访问器属性的 获取函数 与 设置函数
从`源对象` 复制访问器属性 到 `目标对象`时
+ 会先调用该访问器属性的获取函数`get()`
	然后将返回值 作为一个==静态值== 与 该访问器属性的键 组成一个==数据属性== 复制给目标对象
	+ 如果该访问器属性没有获取函数
		则 该访问器属性的键 与 `Undefined`组成一个==数据属性== 复制给目标对象
+ ==该访问器属性的设置函数不会被复制==
```js
dest = {
	set a(val) {
		console.log(`Invoked dest setter with param ${val}`);
	}
};
src = {
	get a() {
		console.log('Invoked src getter');
		return 'foo';
	}
};
Object.assign(dest, src);
// 调用 src 的获取方法
// 调用 dest 的设置方法并传入参数"foo"
// 因为这里的设置函数不执行赋值操作
// 所以实际上并没有把值转移过来
console.log(dest); // { set a(val) {...} }
```
### ==无法回滚==
如果赋值期间出错, 则操作会中止并退出, 同时抛出错误
`目标数组`==不会回到被修改之前的状态==
```js
dest = {};
src = {
	a: 'foo',
	get b() {
		// Object.assign()在调用这个获取函数时会抛出错误
		throw new Error();
	},
	c: 'bar'
};
try {
	Object.assign(dest, src);
} catch(e) {}
// Object.assign()没办法回滚已经完成的修改
// 因此在抛出错误之前，目标对象上已经完成的修改会继续存在：
console.log(dest); // { a: foo }
```
