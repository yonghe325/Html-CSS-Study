[[3. 组合继承|组合继承]]也并非完美
组合继承最主要的问题是效率问题
当子类被创建时时, 父类构造函数会被调用两次
一次是在子类构造函数内部添加属性时调用
一次是创建子类原型时调用父类构造函数
```js
function SuperType(name) {
	this.name = name;
	this.colors = ["red", "blue", "green"];
}
SuperType.prototype.sayName = function() {
	console.log(this.name);
};
function SubType(name, age){
	SuperType.call(this, name); // 子类构造函数内部 SuperType()
	this.age = age;
}
SubType.prototype = new SuperType(); // 为子类创建原型 SuperType()
SubType.prototype.constructor = SubType;
SubType.prototype.sayAge = function() {
	console.log(this.age);
};
```
在上面的代码执行后，`SubType.prototype`上会有两个属性：
	name 和 colors
它们都是 `SuperType` 的实例属性，但现在成为了 `SubType` 的原型属性
在调用 `SubType` 构造函数时，也会调用 `SuperType` 构造函数
这一次会在新对象上创建实例属性 `name` 和 `colors`
导致创建的
这两个实例属性会遮蔽原型上同名的属性

在创建实例后
```js
let instance = new SubType("Nicholas", 29)
```
![[Pasted image 20260415153352.png]]导致有两组`name`,`colors`, 一组在`instance`实例上, 一组在子类`SubType.Prototype`上
这就是调用两次父类构造函数的结果
而寄生式组合继承可以解决这个问题

---
寄生式组合继承通过盗用构造函数继承属性，但使用混合式原型链继承方法
基本思路是不通过调用父类构造函数给子类原型赋值
而是取得父类原型的一个副本
==说到底就是使用寄生式继承来继承父类原型==
==然后将返回的新对象赋值给子类原型==
```js
function inheritPrototype(subType, superType) {
	//					    子类      父类
	let prototype = object(superType.prototype); // 创建对象
		//创建 父类原型的副本
	prototype.constructor = subType; // 增强对象
		//解决字面量修改原型(重写原型),导致constructor属性丢失的问题
		//将其指向 子类的构造函数
	subType.prototype = prototype; // 赋值对象
	//最后将新创建的对象赋值给子类型的原型, 即为父类原型对象的副本
}
```
通过这种方法, 就可以实现只调用一次父类构造函数
	保证子类不继承父类无用的属性
实现子类继承父类, 保证方法可重用
```js
function SuperType(name) {// 父类构造函数
	this.name = name;
	this.colors = ["red", "blue", "green"];
}
SuperType.prototype.sayName = function() {//为父类构造函数的原型添加方法
	console.log(this.name);
};
function SubType(name, age) {//子类构造函数
	SuperType.call(this, name);
	this.age = age;
}

function inheritPrototype(subType, superType) {
	let prototype = object(superType.prototype); 
	prototype.constructor = subType; 
	subType.prototype = prototype; 
}

inheritPrototype(SubType, SuperType);
	//完成寄生组合继承

SubType.prototype.sayAge = function() {
		console.log(this.age);
};
	//在为子类的原型对象添加属性
```
