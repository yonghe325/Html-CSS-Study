# 理解原型
只要创建一个函数, 就会按照特定规则为这个函数创建一个`prototype`属性
	`prototype`属性的值被称为==原型对象==
所有原型对象都会获得一个`constructor`的属性
	`constructor`属性的值==指回与之关联的构造函数==
	即`Person.prototype.constructor`指向`Person`
然后，因构造函数而异，可能会给原型对象添加其他属性和方法
 
---

`Prototype`属性无法通过标准方式访问
但 Firefox、Safari 和 Chrome(edge使用其内核, 自然也会)会在==每个对象==上暴露`__proto__`属性
	==每个对象都有其原型==
通过这个属性可以访问对象的原型
其他内核则完全隐藏该属性
>关键在于理解这一点：
>==实例与构造函数原型之间有直接的联系==，==但实例与构造函数之间没有==
```js
	function Person() {}
	//声明一个构造函数
/**
* 声明之后，构造函数就有了一个
* 与之关联的原型对象：
*/
	console.log(typeof Person.prototype);//Object
	console.log(Person.prototype.constructor === Person); // true
	console.log(Person.prototype);
	// {
		// constructor: f Person(),
			//指向Person构造函数
			//**所以Person构造函数二者相互引用**
		// __proto__: Object
			//原型对象的__proto__属性
			//指向原型对象的原型
	// }
```
# 原型链
更多关于原型链的知识, 转到[[0. 原型链|原型链]]
所以就有这样一条链 : 
实例.`__proto__` -> 构造函数.`prototype` -> 构造函数.`prototype`.`__proto__`
	而这条链, 就被称为==原型链==,==对象的继承就是基于这条链==
正常的原型链都终止与`Object`的原型对象
即为`null`
```js
console.log(Person.prototype.__proto__ === Object.prototype); // true
console.log(Person.prototype.__proto__.constructor === Object); // true
console.log(Person.prototype.__proto__.__proto__ === null); // true
console.log(Person.prototype.__proto__);
// {
	// constructor: f Object(),
	// toString: ...
	// hasOwnProperty: ...
	// isPrototypeOf: ...
	// ...
// }
```
# 构造函数、原型对象和实例之间的关系
![[Pasted image 20260414131116.png]]
构造函数、原型对象和实例,==是 3 个完全不同的对象==：
```js
let person1 = new Person(),
person2 = new Person();
console.log(person1 !== Person); // true
console.log(person1 !== Person.prototype); // true
console.log(Person.prototype !== Person); // true
```
==实例与构造函数原型之间有直接的联系==，==但实例与构造函数之间没有==
因为实例通过`__proto__`链接直接到原型对象, 与构造函数完全没有联系
```js
console.log(person1.__proto__ === Person.prototype); // true
conosle.log(person1.__proto__.constructor === Person); // true
//是原型对象的constructor属性指向对应的构造函数
//与实例之间无联系
```
## 原型对象共享
[[4. 原型层级|实现共享的原理]]
同一个构造函数创建的多个实例, 共享一个原型对象
```js
console.log(person1.__proto__ === person2.__proto__); // true
```
# `instanceof`方法
[[5. 确定类型 instanceof|前面]]提到`instanceof`方法的功能是确定一个对象的类型
而`instanceof`的工作逻辑就是检查
	**右侧构造函数的 `prototype` 是否出现在左侧对象的原型链上**
```js
console.log(person1 instanceof Person); // true
console.log(person1 instanceof Object); // true
console.log(Person.prototype instanceof Object); // true
