>从原型上搜索值的过程是动态的
>所以即使实例在修改原型之前已经存在
>任何时候对 原型对象 所做的修改也 即时的在实例上反映出来
>	因为实例与原型对象之间, 仅靠指针链接
```js
let friend = new Person();
Person.prototype.sayHi = function() {
	console.log("hi");
};
friend.sayHi(); // "hi"，没问题！
```
这就导致了一个问题
可同时有多个原型对象的`constructor`属性只想同一个构造函数
由同一个构造函数创建的对象, 之间的原型对象可能不同
```js
function Person() {}
let friend = new Person();
Person.prototype = {
	constructor: Person,
	name: "Nicholas",
	age: 29,
	job: "Software Engineer",
	sayName() {
		console.log(this.name);
	}
};
//修改Person的原型对象
friend.sayName(); // 错误
//因为此时friend的原型对象还是Person未修改之前的原型对象
```
说明实例的`prototype`属性
不会因为构造函数的原型对象的重写(改变引用)而改变
