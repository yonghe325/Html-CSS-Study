# 构造函数
构造函数用于创建特定类型的对象
原生的`Object`与`Array`这样的就是构造函数
可以通过自定义构造函数, 实现创建对象
```js
function Person(name, age, job){
	this.name = name;
	this.age = age;
	this.job = job;
	this.sayName = function() {
		console.log(this.name);
	};
}
let person1 = new Person("Nicholas", 29, "Software Engineer");
let person2 = new Person("Greg", 27, "Doctor");
person1.sayName(); // Nicholas
person2.sayName(); // Greg
```
构造函数与工厂函数类似, 都有如下区别:
 + 没有显式的创建对象
 + 属性和方法直接赋值个`this`
 + 没有return
---
构造函数的首字母应该大写, 用来区别普通函数与构造函数
构造函数不一定要写成函数声明的形式。赋值给变量的函数表达式也可以表示构造函数
```js
let Person = function(name, age, job) {
	this.name = name;
	this.age = age;
	this.job = job;
	this.sayName = function() {
		console.log(this.name);
	};
}
let person1 = new Person("Nicholas", 29, "Software Engineer");
let person2 = new Person("Greg", 27, "Doctor");
person1.sayName(); // Nicholas
person2.sayName(); // Greg
console.log(person1 instanceof Object); // true
console.log(person1 instanceof Person); // true
console.log(person2 instanceof Object); // true
console.log(person2 instanceof Person); // true
```
---
在实例化时，如果不想传参数，那么构造函数后面的括号可加可不加
只要有 new 操作符，就可以调用相应的构造函数
```js
function Person() {
	this.name = "Jake";
	this.sayName = function() {
		console.log(this.name);
	};
}
let person1 = new Person();
let person2 = new Person;
```
---
+ 构造函数的工作流程
	使用构造函数创建一个对象
	需要使用`new`显式调用构造函数
	当一个构造函数被调用时, 会执行如下操作 : 
	1. 在内存中创建一个新对象
	2. 新对象内部的 `[[Prototype]]` 被赋值为构造函数的 `prototype` 属性
		关于`[[Prototype]]`特性, 转到[[1. 原型模式]]
	3. 构造函数的内部的this被赋值为这个新对象(即this指向新对象)
	4. 执行构造函数内部的代码(给对象添加属性)
	5. 如果构造函数赶回非空对象, 则返回这个对象,==然后新对象被抛弃==
		否则返回这个新创建的对象
---
+ 构造函数 可以 标识对象类型
	关于对象类型, 转到[[5. 确定类型 instanceof|确定对象类型]]
	绝大部分对象都有一个`constructor`
	对于使用构造函数创建的对象
	`constructor`属性指向对应的构造函数本身
	```js
	console.log(person1.constructor == Person); // true
	console.log(person2.constructor == Person); // true
	```
	以这种方式可以判断[[对象标识类型]]
# 构造函数也是函数
>构造函数与普通函数唯一的区别就是调用方式不同
>任何函数使用`new`操作符, 就是构造函数
>而构造函数不适用`new`操作符, 就是普通函数
## 构造函数中的`this`
+ 当构造函数使用`new`调用时
	函数内部的`this`指向新创建的对象
	```js
		// 作为构造函数
		let person = new Person("Nicholas", 29, "Software Engineer");
		person.sayName(); // "Nicholas"
	```
+ 当构造函数作为普通函数调用时
	+ 如果没有使用`call()`或`apply()`显式的进行回调
		转到[[call() 与 apply()]]
		构造函数的内部`this`==默认指向全局(Global)对象==
		在浏览器中, 全局对象为window对象
		所有添加给`this`的属性, 都会被添加给window对象
		```js
		// 作为函数调用
		Person("Greg", 27, "Doctor"); // 添加到 window 对象
		window.sayName(); // "Greg"
		```
	+ 当使用`call()`或`apply()`显式的进行回调, 并且==将特定对象指定为作用域==
		构造函数内部的`this`将指向该特定对象
		所有添加给`this`的属性, 都会被添加给该特定对象
		```js
		// 在另一个对象的作用域中调用
		let o = new Object();
		Person.call(o, "Kristen", 25, "Nurse");
		o.sayName(); // "Kristen"
		```
# 构造函数的问题
例如:
```js
	function Person(name, age, job){
		this.name = name;
		this.age = age;
		this.job = job;
		this.sayName = function() {
			console.log(this.name);
		};
	}
	//上述构造函数在逻辑上如下
	function Person(name, age, job){
		this.name = name;
		this.age = age;
		this.job = job;
		this.sayName = new Function("console.log(this.name)"); // 逻辑等价
	}
```
每个 Person 实例都会有自己的 Function 实例用于显示 name 属性
以这种方式创建函数会带来不同的作用域链和标识符解析
导致不同实例上的函数虽然同名却不相等
```js
	console.log(person1.sayName == person2.sayName); // false
```
因为都是做一样的事，所以没必要定义两个不同的 Function 实例
要解决这个问题，可以把函数定义转移到构造函数外部
```js
function Person(name, age, job){
	this.name = name;
	this.age = age;
	this.job = job;
	this.sayName = sayName;
}
function sayName() {
	console.log(this.name);
}
let person1 = new Person("Nicholas", 29, "Software Engineer");
let person2 = new Person("Greg", 27, "Doctor");
person1.sayName(); // Nicholas
person2.sayName(); // Greg
```
此时`this.sayName = sayName`中的属性值仅为指向外部的一个指针
这样虽然解决了相同逻辑的函数重复定义的问题
但全局作用域也因此被搞乱了，因为那个函数实际上只能在一个对象上调用
如果这个对象需要多个方法
那么就要在全局作用域中定义多个函数
这会导致自定义类型引用的代码不能很好地聚集一起
这个新问题可以通过[[1. 原型模式|原型模式]]来解决
