类的语法可以非常方便地定义应该存在于实例上的成员
应该存在于原型上的成员，以及应该存在于类本身的成员
# 实例成员
每次通过new调用类标识符时，都会执行类构造函数
在这个函数内部，可以为新创建的实例（`this`）添加“自有”属性
至于添加什么样的属性，则没有限制
另外，在构造函数执行完毕后，仍然可以给实例继续添加新成员
即在 类构造函数外部, 类块内部, 为实例添加新属性
	相当于为原型添加属性
每个实例都对应一个唯一的成员对象，这意味着所有成员都不会在原型上共享
```js
class Person {
	constructor() {
	// 这个例子先使用对象包装类型定义一个字符串
	// 为的是在下面测试两个对象的相等性
	this.name = new String('Jack');
	this.sayName = () => console.log(this.name);
	this.nicknames = ['Jake', 'J-Dog']
	}
}
let p1 = new Person(),
p2 = new Person();
p1.sayName(); // Jack
p2.sayName(); // Jack
console.log(p1.name === p2.name); // false
console.log(p1.sayName === p2.sayName); // false
console.log(p1.nicknames === p2.nicknames); // false
p1.name = p1.nicknames[0];
p2.name = p2.nicknames[1];
p1.sayName(); // Jake
p2.sayName(); // J-Dog
```
# 原型方法与访问器
在类构造函数内部添加的方法会被直接赋值给每一个实例
	不同实例之间不会共用
而在==类块中定义的方法==会被所有实例引用
	相当于添加在原型上的方法
这意味着如果在 类构造函数 与 类块中定义相同的属性
类块中定义的属性 会 遮蔽 类构造函数中定义的属性
```js
class Person {
	constructor() {
		// 添加到 this 的所有内容都会存在于不同的实例上
		this.locate = () => console.log('instance');
	}
// 在类块中定义的所有内容都会定义在类的原型上
	locate() {
		console.log('prototype');
	}
}
let p = new Person();
p.locate(); // instance
Person.prototype.locate(); // prototype
```
==但不能在类块中给原型添加原始值或对象作为成员数据==
	**==已经过时了==**, ECMA2022之后允许在类块内部 添加类字段属性
	此时该变量会成为不会被实例之间共享的实例属性, 而不是原型属性
	原型属性仍然需要 在类块外部定义
	具体转到[[3 实例、原型和类成员#类字段属性|类字段属性]]
```js
class Person {
	name: 'Jake'
}
// Uncaught SyntaxError: Unexpected token
```
类方法等同于对象属性，因此可以使用字符串、符号或计算的值作为键
```js
const symbolKey = Symbol('symbolKey');
class Person {
	stringKey() {
		console.log('invoked stringKey');
	}
	[symbolKey]() {
		console.log('invoked symbolKey');
	}
	['computed' + 'Key']() {
		console.log('invoked computedKey');
	}
}
let p = new Person();
p.stringKey(); // invoked stringKey
p[symbolKey](); // invoked symbolKey
p.computedKey(); // invoked computedKey
```
类定义也支持获取和设置访问器属性
语法与行为跟普通对象一样
```js
class Person {
	set name(newName) {
		this.name_ = newName;
	}
	get name() {
		return this.name_;
	}
}
let p = new Person();
p.name = 'Jake';
console.log(p.name); // Jake
```
# 静态类方法
可以在类上定义静态方法
这些方法通常用于执行不特定于实例的操作，也不要求存在类的实例
静态成员每个类上只能有一个
使用类静态方法相当于使用挂到该类上的一个函数
且对应的静态方法只能通过该类来调用
```js
class Person {
	constructor() {
	// 添加到 this 的所有内容都会存在于不同的实例上
		this.locate = () => console.log('instance', this);
	}
	// 定义在类的原型对象上
	locate() {
		console.log('prototype', this);
	}
	// 定义在类本身上
	static locate() {
		console.log('class', this);
	}
}
let p = new Person();
p.locate(); // instance, Person {}
Person.prototype.locate(); // prototype, {constructor: ... }
Person.locate(); // class, class Person {}
```
静态类方法非常适合作为实例工厂
```js
class Person {
	constructor(age) {
		this.age_ = age;
	}
	sayAge() {
		console.log(this.age_);
	}
	static create() {
		// 使用随机年龄创建并返回一个 Person 实例
		return new Person(Math.floor(Math.random()*100));
	}
}
console.log(Person.create()); // Person { age_: ... }
```

## 概述
类字段（Class Fields）是 ES2022（ES13）正式纳入标准的特性，允许在类块中直接定义实例属性、静态属性和私有属性，无需在构造函数中通过 `this` 添加。
类字段会在调用构造函数之前执行初始化

---
## 基本语法
```javascript
class MyClass {
    // 实例公有字段
    instanceField = '默认值';
    
    // 静态公有字段
    static staticField = '静态值';
    
    // 私有字段（以 # 开头）
    #privateField = '私有值';
    
    // 静态私有字段
    static #privateStaticField = '静态私有值';
}
```
## 实例公有字段
实例之间不共享
```js
class Person {
    // 实例字段（每个实例独立）
    name = 'Unknown';
    age = 0;
    friends = [];
    constructor(name, age) {
        // 可以覆盖字段的默认值
        this.name = name;
        this.age = age;
    }
    addFriend(friend) {
        this.friends.push(friend);
    }
}
const p1 = new Person('Alice', 25);
const p2 = new Person('Bob', 30);
p1.addFriend('Charlie');
console.log(p1.friends);  // ['Charlie']
console.log(p2.friends);  // []（不受影响）
console.log(p1.name);  // 'Alice'
console.log(p2.name);  // 'Bob'
```
## 静态属性
实例无法访问静态字段, 只能通过类来访问
==在类继承中, 静态公有字段可以被遮蔽==
	转到[[4. 继承|类继承]]
```js
class Config {
    // 静态字段（属于类本身，所有实例共享）
    static appName = 'MyApp';
    static version = '1.0.0';
    static apiUrl = 'https://api.example.com';
    
    constructor() {
        // 可以通过类名访问
        console.log(`App: ${Config.appName}`);
    }
    
    static getConfig() {
        // 静态方法中可以用 this 访问静态字段
        return {
            name: this.appName,
            version: this.version,
            url: this.apiUrl
        };
    }
}

// 通过类名访问静态字段
console.log(Config.appName);  // 'MyApp'
console.log(Config.version);  // '1.0.0'

// 静态方法调用
console.log(Config.getConfig());

// 实例无法访问静态字段
const c = new Config();
console.log(c.appName);  // undefined
```
## 私有属性
私有字段使用 `#` 前缀，只能在类的内部访问，外部无法读取或修改
+ 私有字段是硬私有, 无法通过任何方式从外部访问
+ ==私有属性无法删除==
+ 私有子段可以 与 同名公有字段同时存在
+ 私有字段不会被继承, 子类无法访问父类的私有字段
```js
class BankAccount {
    // 私有字段
    #balance = 0;
    #accountNumber;
    #transactions = [];
    constructor(accountNumber, initialBalance) {
        this.#accountNumber = accountNumber;
        this.#balance = initialBalance;
    }
    // 公有方法访问私有字段
    deposit(amount) {
        if (amount > 0) {
            this.#balance += amount;
            this.#addTransaction('deposit', amount);
            return true;
        }
        return false;
    }
    withdraw(amount) {
        if (amount > 0 && amount <= this.#balance) {
            this.#balance -= amount;
            this.#addTransaction('withdraw', amount);
            return true;
        }
        return false;
    }
    getBalance() {
        return this.#balance;
    }
    // 私有方法
    #addTransaction(type, amount) {
        this.#transactions.push({
            type,
            amount,
            date: new Date()
        });
    }
    // 通过 getter 暴露交易记录（只读）
    get transactions() {
        return [...this.#transactions];  // 返回副本，防止外部修改
    }
}
const account = new BankAccount('123456', 1000);
account.deposit(500);
account.withdraw(200);
console.log(account.getBalance());  // 1300
```
无法直接访问私有字段
```js
// console.log(account.#balance);      // SyntaxError
// console.log(account.#accountNumber); // SyntaxError
// account.#addTransaction();           // SyntaxError
```
可以通过提供的公有方法间接访问私有属性
```js
console.log(account.transactions);
// [{ type: 'deposit', amount: 500, date: ... },
//  { type: 'withdraw', amount: 200, date: ... }]
```

# 非函数原型和类对象
虽然类定义并不显式支持在原型或类上添加成员数据
但在类定义外部，可以手动添加
	==**已过时**==, 具体转到[[3 实例、原型和类成员#类字段属性]]
```js
class Person {
	sayName() {
		console.log(`${Person.greeting} ${this.name}`);
	}
}
// 在类上定义数据成员
Person.greeting = 'My name is';
// 在原型上定义数据成员
Person.prototype.name = 'Jake';
let p = new Person();
p.sayName(); // My name is Jake
```
