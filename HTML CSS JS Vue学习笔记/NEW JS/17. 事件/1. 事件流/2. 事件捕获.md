Netscape Communicator 团队提出了另一种名为事件捕获的事件流
事件捕获的意思是最不具体的节点应该最先收到事件，而最具体的节点应该最后收到事件
	与事件冒泡完全相反
事件捕获实际上是为了在事件到达最终目标前拦截事件
如果前面的例子(事件冒泡的例子)使用事件捕获，则点击\<div>元素会以下列顺序触发 click 事件
1.  document
 2. \<html>
 3. \<body>
 4. \<div>

事件捕获得到了所有现代浏览器的支持。
实际上，所有浏览器都是从 window 对象开始捕获事件
而 DOM2 Events规范规定的是从 document 开始

旧版本浏览器不支持事件捕获，因此实际当中几乎不会使用事件捕获
特殊情况下可以使用事件捕获
