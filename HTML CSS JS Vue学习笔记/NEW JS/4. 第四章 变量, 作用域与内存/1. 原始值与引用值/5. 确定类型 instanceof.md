使用`typeof`适合可以判断一个变量是否为原始类型
但当`typeof`遇到`null`, `Object`类型时
他只会返回`Object`
```js
let s = "Nicholas";
let b = true;
let i = 22;
let u;
let n = null;
let o = new Object();

console.log(typeof s); // string
console.log(typeof i); // number
console.log(typeof b); // boolean
console.log(typeof u); // undefined
console.log(typeof n); // object
console.log(typeof o); // object
```
表明`typeof`对原始值可以直接判断出
但对引用类型作用不大
所有引用类型都是对象, `typeof`并不能确定一个引用值是何种类型的对象
# instanceof操作符
[[1. 原型模式#`instanceof`方法|instanceof的底层逻辑]]
`instanceof`与`typeof`的功能类似
`instanceof`的功能是检查一个对象是否为指定的 ==对象类型==
因此不适合检查原始类型
```js
result = variable instanceof constructor

console.log(person instanceof Object); // 变量 person 是 Object 吗？
console.log(colors instanceof Array); // 变量 colors 是 Array 吗？
console.log(pattern instanceof RegExp); // 变量 pattern 是 RegExp 吗？
```
`instanceof`有一系列与之类似的检查函数
+ `Array.isArray()` 专门检查数组
+ `Object.prototype.toString.call()` 返回该对象的类型
	```js
	console.log(Object.prototype.toString.call(  []  ));      // "[object Array]"
	console.log(Object.prototype.toString.call(  {}  ));      // "[object Object]"
	console.log(Object.prototype.toString.call(  null  ));    // "[object Null]"
	```
