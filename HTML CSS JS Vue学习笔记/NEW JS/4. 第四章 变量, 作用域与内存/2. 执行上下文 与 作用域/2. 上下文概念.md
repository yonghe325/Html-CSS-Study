上下文是一个概念, 决定了变量或者函数可以访问那些数据
>每个上下文都有一个关联的变量对象(variable object)
>而这个上下文中定义的所有变量和函数都存在于这个对象上
>虽然无法通过代码访问变量对象，但后台处理数据会用到它

简单来说, 上下文决定了`this`指向的对象
>每个上下文都可以到上一级上下文中去搜索变量和函数
>但任何上下文都不能到下一级上下文中去搜索
# 执行上下文也分层级
执行上下文包括 : 全局上下文, 函数上下文 和 块级上下文
+ 全局上下文是最外层的上下文
	ECMAScript运行的环境(宿主环境)决定了全局上下文的对象
	+ 在浏览器中, 全局上下文就是 window 对象
>使用 let 和 const 的顶级声明不会定义在全局上下文中, 但在作用域链解析上效果是一样的
>上下文在其所有代码都执行完毕后会被销毁，包括定义在它上面的所有变量和函数
>（全局上下文在应用程序退出前才会被销毁，比如关闭网页或退出浏览器）
# 上下文 与 作用域链的关系
```js
执行上下文（ExecutionContext）
├── 变量环境（Variable Environment）  ← 当前作用域
├── 词法环境（Lexical Environment）   ← 当前作用域（含块级）
├── 作用域链（Scope Chain）           ← 查找路线图
└── this 绑定（This Binding）         ← 当前对象指向
```

>==代码执行流每进入一个新上下文，都会创建一个作用域链，用于搜索变量和函数==

| 特性       | 执行上下文（Context） | 作用域链（Scope Chain） |
| -------- | -------------- | ----------------- |
| **本质**   | 函数调用时的环境       | 变量查找的路径           |
| **创建时机** | 函数调用时          | 函数定义时             |
| **包含内容** | 变量对象、this、作用域链 | 变量对象的引用链          |
| **生命周期** | 函数执行期间         | 函数存在期间（闭包）        |
| **数量关系** | 每次调用都创建新上下文    | 每个函数只有一条作用域链      |
| **修改性**  | 每次调用都新创建       | 定义后固定（除with/eval） |
| **与闭包**  | 执行完可能被销毁       | 闭包保持作用域链活跃        |
```js
const obj = {
    name: "Object",
    log: function() {
        console.log(this.name);  // 问this：如何调用？
        console.log(name);       // 问作用域链：name在哪里定义？
    }
};
const name = "Global";  // 全局作用域
obj.log();  
// 输出：Object（this指向obj）
// 输出：Global（作用域链找到全局name）
const log = obj.log;
log();
// 输出：undefined（this指向全局）
// 输出：Global（作用域链不变）
```
