# var函数作用域声明
当使用`var`进行变量声明时, 该变量会自动添加到最近的上下文中
在函数中声明, 则添加到最近的函数上下文中
==当一个变量未经声明就被初始化==
==则该变量会被添加到全局上下文中==
```js
function add(num1, num2) {
	sum = num1 + num2;
	return sum;
}
let result = add(10, 20); // 30
console.log(sum); // 30
```

# let的块级作用域声明
块级作用域由`{ }`界定
只要是语法承认的`{ }`, 都是块级作用域
其实`let`与`var`都会进行**提升**
### let的提升
提升分为 ==值提升== 和 ==位置提升==
`var`就是 值提升 + 位置提升
而`let`只有 位置提升
简单来说就是 : 
+ 当一个变量使用`var`或`let`进行声明时
	==都会将声明提升到**块作用域**的顶部==
	但`var`会将该变量同时赋值为`undefined`
	而`let`不会, 所以在声明前使用`let`声明的变量
	不会输出**未找到**的异常
	而是输出**未初始化**的异常
	但如果在声明前使用`let`声明的变量后, 没有声明该变量
	则会输出**未找到**的异常
	```js
	var c = "outer";
	function test() {
	    // 如果 let 没有提升，这里应该打印 "outer"
	    // 但实际上：
	    console.log(c);  // ❌ 报错！Cannot access 'c' before initialization
	    let c = "inner";
	}
	```
# const的常量声明时
使用`const`声明的变量
必须在声明时进行初始化
且在该变量的生命周期内不能再赋值
对于使用`const`声明的引用变量, 键值是可以改变的, 但不能再重新被赋值为其他引用值
[[3. const]]
如果想让对象整体都不能被修改
可以只用`Object.freezr(<对象>)`函数进行处理
此时该对象将不能再被修改
