==当一个作用域产生/声明/定义时, 就会对应产生该作用域的作用域链==
*(其实是创建对应的执行上下文)*
使用一个变量时, 会在所在作用域查找该变量, 如果没有找到
则向上级作用域查找(即从内向外查找)
而这个查找的过程所经过的作用域的路径
叫做作用域链 ^1af7fd
# 作用域链的本质
作用域链的本质是一个链表结构每个节点是一个[[JS 术语#词法环境记录(Lexical Environment Record)|词法环境记录]]
每个指针分别指向一个变量对象
但物理上并不会包含相应的对象
```js
每个节点{
	environmentRecord: { /* 当前作用域的变量 */ },
	outer: <指向父环境记录的引用> 
	// 这就是作用域链
}
```
# 静态作用域
简单来说就是 : 
一个函数的作用域链 不会随着该函数调用时的位置改变 而 改变
```js
var name = "global";
function outer() {
    var name = "outer";
    function inner() {
        console.log(name);  // 输出什么？
    }
    return inner;
}
// 场景1：正常调用
const innerFunc = outer();
innerFunc();  // "outer" ✓
// 场景2：在全局调用  
function callAtGlobal(fn) {
    fn();
}
callAtGlobal(innerFunc);  // 还是"outer"！不是"global"！
// ✅ 关键：作用域链在函数**定义时**确定，不是调用时！
// inner函数的作用域链：[inner] → [outer] → [global]
// 无论在哪调用，都去outer作用域找name
```

# 函数调用时的作用域链活动
在调用一个函数时，会为这个函数调用创建一个执行上下文并创建一个作用域链
	转到[[2. 上下文]]
然后用 `arguments`和其他命名参数来初始化这个函数的活动对象
	转到[[3. 理解参数|函数参数]] 与 [[5. 默认参数值#默认参数作用域 与 暂时性死区|默认参数作用域]]
外部函数的活动对象 是 内部函数作用域链上 的第二个对象
这个作用域链一直向外串起了所有包含函数的活动对象
直到全局执行上下文才终止
