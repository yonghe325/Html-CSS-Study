标记清理（mark-and-sweep）, 最常用的垃圾回收策略
# 标记阶段
当在标记阶段时
清理程序会从 [[JS 术语#根对象（Roots）|根对象]]开始, 递归标记所有[[JS 术语#可达性（Reachability）|可达对象]]
加上标记的变量从逻辑上讲, 永远不会被清理
*做标记的方式 :* 
标记时可以采用各种不影响变量本身的方式
可以对变量的某一位进行取反, 可以单独维护一个数组
标记只是为了标识变量, 所以标记的过程不重要, 重要的是如何实现标记的清理
所以标记的方式有很多种
# 清理阶段 
清理程序会将==所有在上下文中的变量==，以及==被在上下文中的变量引用的变量的标记==去掉
在此之后再被加上标记的变量就是待删除的了，原因是任何在上下文中的变量都访问不到它们了
随后垃圾回收程序做一次内存清理，==销毁带标记的所有值并收回它们的内存==
