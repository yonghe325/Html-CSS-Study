为每一个==值==记录它被==引用==的次数
当声明变量并赋给它一个引用值时, 这个值的引用次数就 +1 
相应的, 当被赋值的变量被其他值覆盖时, 这个值的引用次数就 -1
当一个值的引用数为 0 时, 没办法使用这个值了
垃圾回收程序就可以将其内存回收
# 问题
## 性能问题
每一次引用一个值, 都会对引用次数进行更新, 性能消耗大
## 循环引用
引用计数可以应用于原始类型的值
但对引用类型来说, 可能会出现==循环引用==的问题
```js
function problem() {
	let objectA = new Object();
	let objectB = new Object();
	objectA.someOtherObject = objectB;
	objectB.anotherObject = objectA;
}
```
在该函数作用域内`objectA`与`objectB`同时被对方引用
导致两个变量的值的引用次数都为 2 , 且离开上下文后由于仙湖引用的相互引用的缘故
二者的引用次数永远都不会变为 0 , 导致二值永远不会被清理
==这是严重内存泄露问题==
## DOM 与 JavaScript 的跨边界循环引用
这是更隐蔽的 循环引用的 情况[ds]
```js
// 更隐蔽的内存泄漏
function 绑定事件() {
    var element = document.getElementById("myButton");
	    element.onclick = function() {
        console.log("Clicked");
        // 这个函数引用了element（闭包）
        // element也通过onclick引用了这个函数
        // 循环引用！
	    };
    // 即使移除DOM元素，由于循环引用也无法回收
}
// IE 6-7 中这会永久泄漏内存
```
