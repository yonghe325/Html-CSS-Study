# 优化手段
## 解除引用
当一个数据不再被使用, 就可以把它覆盖为`null`, 从而释放其引用
这种手段就是 ==显式解除引用==
适合全局变量 和 全局变量的属性 
而局部变量在超出作用域后会被自动解除引用(==隐式解除引用==)
```js
function createPerson(name){
	let localPerson = new Object();
	localPerson.name = name;
	return localPerson;
}
let globalPerson = createPerson("Nicholas");
// 解除 globalPerson 对值的引用
globalPerson = null;
```
解除引用并不能将其内存回收
==解除引用的关键在于确保相关的值已经不在上下文里了, 因此它在下次垃圾回收时会被回收==
## 通过`const` 和 `let`声明提升性能
`const` 和 `let`都是块作用域
可以尽早让垃圾清理程序介入, 回收其内存
## 隐藏类 和 删除操作
不同的浏览器使用不同的[[JS 术语#JavaScript引擎|JavaScript引擎]]
主流的 V8引擎在编译阶段时, 会使用 ==隐藏类== 这一优化方案
简单来说, 就是使用同一个 构造函数 和  创建的多个对象 与 隐藏类相关联
能够共享相同隐藏类的对象性能会更好
```js
function Article() {
	this.title = 'Inauguration Ceremony Features Kazoo Band';
}
let a1 = new Article();
let a2 = new Article();
```
V8 会在后台配置，让这两个类实例共享相同的隐藏类
因为这两个实例共享同一个构造函数和原型
而对 共用隐藏类对象 的==修改==(增删), 会使其不再共用相同的隐藏类
```js
a2.author = 'Jake';
//如果a2在之后新添加了author属性
//这会导致 Article()实例(a1,a2) 对应两个不同的隐藏类
```
隐藏类的大小 与 对共用隐藏类对象的修改(==动态属性赋值==, 当然也包括删除属性)
会对性能产生明显的影响
### 解决方法
+ 对于添加属性
	避免JavaScript的"先创建再补充"(ready-fire-aim)式 的 ==动态属性赋值==
	==尽量在构造函数中一次性声明所有属性==
+ 对于删除属性
	将不需要的属性赋值为`null`
	避免使用`delete`方式, 删除属性, 因为这会脱离 隐藏类共享
	该方法同时还能达到删除引用值供垃圾回收程序回收的效果
