# 未闭包调用时
简单对下列活动总结:
1. 定义时 : 创建函数对象, 确定作用域链(`[[Scope]]`存储 VO)
2. 调用时 : 创建执行上下文, 通过复制`[[Scope]]`复制作用域链, 创建AO, 推入
3. 执行时 : 从作用域链中查找变量, 执行代码
4. 返回后 : 销毁执行上下文 和 活动对象AO
##  函数 定义/调用/执行 时的 作用域链/上下文 活动
```js
function compare(value1, value2) {
	if (value1 < value2) {
		return -1;
	} else if (value1 > value2) {
		return 1;
	} else {
		return 0;
	}
}

let result = compare(5, 10);
```
### 定义时
在定义`compare()`函数时
就会为它创建作用域链，==预装载==全局变量对象
并保存在内部的`[[Scope]]`中
### 调用时
`compare()`函数是在全局上下文中调用的
在调用时 , ==会创建相应的执行上下文==
然后通过复制函数的`[[Scope]]`来创建其作用域链

---
第一次调用`compare()`时
会为它创建一个包含`arguments`, `value1`与`value2`的 **活动对象**(见[[1. 闭包#执行时|执行时]])
	并将其推入作用域链的前端, 成为其作用域链上的第一个对象
全局上下文的 **变量对象**(见[[1. 闭包#执行时|执行时]]) 则是 `compare()`作用域链上的第二个对象
	其中包含`this`, `result`, `compare`
这意味着 `compare()`函数执行上下文的作用域链中有两个变量对象：
+ 局部变量对象 -> 活动对象(AO)
+ 全局变量对象 -> 变量对象(VO)

### 执行时
函数执行时，每个执行上下文中都会有一个包含其中变量的对象
全局上下文中的叫 ==**变量对象(Variable Object (VO))**==，它会在代码执行期间始终存在
而函数局部上下文中的叫 ==**活动对象(Activation Object (AO))**==，只在函数执行期间存在
函数内部的代码在访问变量时，就会使用给定的名称从作用域链中查找变量
函数执行完毕后，==局部活动对象会被销毁==，内存中就只剩下全局作用域

## 不使用闭包的问题
当函数嵌套调用时
```js
function inner() {
    console.log(outerVar);
}

function outer() {
    let outerVar = 'outer';
    inner();  
}

outer();  // 异常
```
内层函数 与 外层函数 在同意作用域下声明是
内层函数无法直接使用外层函数中的变量
因为在内层函数在定义时, `[[Scpoe]]`只有外层作用域的变量
并没有外层函数内部的变量
### 解决
此时, 可以使用传参解决
```js
function inner(outerVar) {
    console.log(outerVar);
}
function outer() {
    let outerVar = 'outer';
    inner(outerVar);  // 传递参数
}
outer();  // 'outer'
```
或是直接在作用域下定义变量(不推荐)
```js
let sharedVar;
function inner() {
    console.log(sharedVar);
}

function outer() {
    sharedVar = 'outer';
    inner();
}
outer();  // 'outer'
```
或是直接 将内层函数移到外层函数内部定义
也就是闭包
```js
function outer() {
    let outerVar = 'outer';
    function inner() {
        console.log(outerVar);
    }
    inner();
}
outer();  // 'outer'
```

# 使用闭包时
```js
function createComparisonFunction(propertyName) {
	return function(object1, object2) {
		let value1 = object1[propertyName];
		let value2 = object2[propertyName];
		if (value1 < value2) {
			return -1;
		} else if (value1 > value2) {
			return 1;
		} else {
			return 0;
		}
	};
}
```
内部定义的函数只有在外层函数执行到内层函数定义的代码时
内层函数才会开始通过复制`[[Scope]]`创建作用域链
在一个函数内部定义的函数会把其 包含函数(外部函数)的活动对象添加到自己的作用域链中
	因为 内层函数 的 `[[Scope]]`是通过 复制外层函数的`[[Scope]]`创建的
	此时外层函数已经将AO推入`[[Scope]]`
	所以 内层函数 作用域链中, 会有 外部函数 的活动对象
因此，在`createComparisonFunction()`函数中
匿名函数的作用域链中实际上包含 `createComparisonFunction()`的活动对象(AO)
之后执行以下代码
```js
let compare = createComparisonFunction('name');
let result = compare({ name: 'Nicholas' }, { name: 'Greg' });
```
![[Pasted image 20260419143658.png]]
在 `createComparisonFunction()`返回匿名函数后
它的作用域链被初始化为包含 `createComparisonFunction()` 的活动对象 和 全局变量对象
## 使用闭包的副作用
`createComparisonFunction()`的活动对象并不能在它执行完毕后销毁
==因为匿名函数的作用域链中仍然有对它的引用==
`createComparisonFunction()`执行完毕后
其执行上下文的作用域链会销毁
但它的活动对象仍然会保留在内存中，直到匿名函数被销毁后才会被销毁
	==也就是没有东西引用活动变量时, 才会销毁==
```js
// 创建比较函数
let compareNames = createComparisonFunction('name');
// 调用函数
let result = compareNames({ name: 'Nicholas' }, { name: 'Matt' });
// 解除对函数的引用，这样就可以释放内存了
compareNames = null;
//把 compareNames 设置为等于 null 会解除对函数的引用
```
## 使用闭包的代价
>因为闭包会保留它们包含函数的作用域，所以比其他函数更占用内存
过度使用闭包可能导致内存过度占用，因此建议仅在十分必要时使用
	转到
>V8 等优化的 JavaScript 引擎会努力回收被闭包困住的内存
不过我们还是建议在使用闭包时要谨慎。
