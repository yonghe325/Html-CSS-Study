在 ECMAScript 5 中，函数内部存在两个特殊的对象：`arguments` 和 `this`
ECMAScript 6 又新增了 `new.target` 属性
# `arguments`对象
除开前文所说的`arguments`的各种特点
	转到[[3. 理解参数#`arguments`对象|arguments对象]],[[3. 理解参数#箭头函数中的参数|箭头函数中的arguments对象]]
`arguments`对象还有一个 `callee` 属性
该属性是一个指向 `arguments` 对象所在函数的指针
## `callee`属性
下面这个经典的阶乘函数:
```js
function factorial(num) {
	if (num <= 1) {
		return 1;
	} else {
		return num * factorial(num - 1);
	}
}
```
在 `factorial` 函数内部直接指定了递归调用的函数, 即`factorial`本身
只要给函数一个名称，而且这个名称不会变，这样定义就没有问题
但是要保证函数能够正确执行, 就必须保证函数名是`factorial`
	==从而导致 函数逻辑 与 函数名 紧密耦合==
而使用`arguments.callee`属性, 就可以事项函数内部递归调用时的函数逻辑 与 函数名 解耦
```js
function factorial(num) {
	if (num <= 1) {
		return 1;
	} else {
		return num * arguments.callee(num - 1);
	}
}
```
这个重写之后的 `factorial()` 函数已经用 `arguments.callee` 代替了之前硬编码的 `factorial`
这意味着无论函数叫叫什么名字, 都可以正确的执行
```js
function factorial(num) {
	if (num <= 1) {
		return 1;
	} else {
		return num * arguments.callee(num - 1);
	}
}
let trueFactorial = factorial;
factorial = function() {
	return 0;
};
	//这里虽然重写了factorial对象
	//但是trueFactorial变量引用的还是之前的函数地址
	//所以使用trueFactorial访问的是之前的阶乘函数
	//使用factorial访问的是修改之后的return 0;的函数
console.log(trueFactorial(5)); // 120
	//如果不适用arguments.callee属性, 
	//而是直接硬编码, 阶乘函数会返回0
	//因为内部调用修改之后的factorial函数, 直接返回了0
console.log(factorial(5)); // 0
```

# `this`对象
| 调用方式                        | `this` 指向                         |
| --------------------------- | --------------------------------- |
| 普通函数调用 `fn()`               | 非严格模式：`window` / 严格模式：`undefined` |
| 对象方法调用 `obj.fn()`           | 对象本身                              |
| 构造函数调用 `new fn()`           | 新创建的实例                            |
| 通过 `call`/`apply`/`bind` 调用 | 指定的对象                             |
`this`对象在 箭头函数 与 标准函数 中有不同的行为
	关于箭头函数中的`this`, 转到[[1. 箭头函数#箭头函数与普通函数的区别]]
## `this`在标准函数中
在标准函数中，`this` 引用的是把函数当成方法调用的上下文对象
这时候通常称其为 `this` 值 (在网页的全局上下文中调用函数时，`this` 指向 `windows`)
来看下面的例子
```js
window.color = 'red';
let o = {
	color: 'blue'
};
function sayColor() {
	console.log(this.color);
}
sayColor(); // 'red'
o.sayColor = sayColor;
	//为对象o动态赋值一个方法
	//键为sayColor, 值为指向sayColor函数的一个指针
o.sayColor(); // 'blue'
```
## `this`在箭头函数中
在箭头函数中，`this` 引用的是==定义==箭头函数的上下文
```js
window.color = 'red';
let o = {
	color: 'blue'
};
let sayColor = () => console.log(this.color);
	//该箭头函数在全局上下文中定义
	//所以内部的this指向全局对象window
sayColor(); // 'red'
o.sayColor = sayColor;
o.sayColor(); // 'red'
```
## 使用`this`回调
>有读者知道，在事件回调或定时回调中调用某个函数时
`this` 值指向的并非想要的对象
此时将回调函数写成箭头函数就可以解决问题
这是因为箭头函数中的 `this` 会保留定义该函数时的上下文
```js
function King() {
	this.royaltyName = 'Henry';
	// this 引用 King 的实例
	setTimeout(() => console.log(this.royaltyName), 1000);
}
function Queen() {
	this.royaltyName = 'Elizabeth';
	// this 引用 window 对象
	setTimeout(function() { console.log(this.royaltyName); }, 1000);
}
new King(); // Henry
new Queen(); // undefined
```
# `caller`属性
ECMAScript 5 也会给函数对象上添加一个属性：`caller`
虽然 ECMAScript 3 中并没有定义，但所有浏览器除了早期版本的 Opera 都支持这个属性
这个属性引用的是==调用当前函数的函数==
或者如果是在全局作用域中调用的则为 null
```js
function outer() {
	inner();
}
function inner() {
	console.log(inner.caller);
}
outer();//[Function: outer]
//ourter() 调用了 inner()，inner.caller 指向 outer()
```
如果要降低耦合度，则可以通过 `arguments.callee.caller` 来引用同样的值
```js
function outer() {
	inner();
}
function inner() {
	console.log(arguments.callee.caller);
	//arguments.callee指向inner函数
	//所以arguments.callee.caller只想调用inner函数的函数
	//arguments.callee.caller() == outer()
}
outer();//[Function: outer]
```

+ ECMAScript 5 也定义了 `arguments.caller`
		严格模式下等价于`arguments.callee.caller`
		在非严格模式下使用则抛出异常
	+ 在ECMAScript 6中, 已经被废弃
		严格模式下访问它会报错，在非严格模式下则始终是 `undefined`
+ 在严格模式下, 不能给函数的 `caller` 属性赋值, 否则抛出异常

# `new.target`属性
ECMAScript 中的函数(除了箭头函数)始终可以作为构造函数实例化一个新对象
检测函数是否使用 `new` 关键字调用的 `new.target` 属性
如果函数是正常调用的
则`new.target`的值为`Undefined`
如果函数是通过`new`关键字调用的
则`new.target`的将引用被调用的构造函数
```js
function King() {
	if (!new.target) {
		throw 'King must be instantiated using "new"'
	}
		console.log('King instantiated using "new"');
	}
new King(); // King instantiated using "new"
King(); // Error: King must be instantiated using "new"
```
**使用 `new` 调用函数时，无论 `new.target` 是什么值，都会创建新对象
`new.target` 只是用来检测函数是否被 `new` 调用
然后在执行相关的操作
而不是决定是否创建对象**
