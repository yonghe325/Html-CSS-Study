尾调用优化的条件就是确定外部栈帧真的没有必要存在了
涉及的条件如下:
+ 代码在严格模式下执行
	>之所以要求严格模式
	主要因为在非严格模式下函数调用中允许使用 `f.arguments`和 `f.caller`
	而它们都会引用外部函数的栈帧, 显然, 这意味着不能应用优化了
	因此尾调用优化要求必须在严格模式下有效，以防止引用这些属性
+ 外部函数的返回值是对尾调用函数的调用
	```js
	// 无优化：尾调用没有返回
	function outerFunction() {
		innerFunction();
	}
	```
+ 尾调用函数返回后不需要执行额外的逻辑
	```js
	// 无优化：尾调用返回后必须转型为字符串
	function outerFunction() {
		return innerFunction().toString();
	}
	```
+ 尾调用函数不是引用外部函数作用域中自由变量的闭包
	具体转到[[1. 闭包|闭包]]
	```js
	// 无优化：尾调用是一个闭包
	function outerFunction() {
		let foo = 'bar';
		function innerFunction() { return foo; }
			return innerFunction();
		}
	}
	```
## 以下情况有优化
```js
"use strict";
// 有优化：栈帧销毁前执行参数计算
function outerFunction(a, b) {
	return innerFunction(a + b);
}
// 有优化：初始返回值不涉及栈帧
function outerFunction(a, b) {
	if (a < b) {
		return a;
	}
	return innerFunction(a + b);
}
// 有优化：两个内部函数都在尾部
function outerFunction(condition) {
	return condition ? innerFunctionA() : innerFunctionB();
}
```
# 尾调用优化差异?
差异化尾调用和递归尾调用是容易让人混淆的地方
无论是递归尾调用还是非递归尾调用，都可以应用优化
引擎并不区分尾调用中调用的是函数自身还是其他函数
不过，这个优化在递归场景下的效果是最明显的
因为递归代码最容易在栈内存中迅速产生大量栈帧
