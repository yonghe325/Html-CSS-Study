递归函数通常的形式是一个函数通过名称调用自己
```js
function factorial(num) {
	if (num <= 1) {
		return 1;
	} else {
		return num * factorial(num - 1);
	}
}
```
在非严格模式下,通过使用`arguments.callee`
可以实现递归函数内部调用的解耦
转到[[9. 函数内部#`arguments`对象#`callee`属性|callee属性的使用]]
但是在严格模式下, `arguments.callee`禁止访问, 访问会出错
此时，可以使用 ==**命名函数表达式**（named function expression）==达到目的
# 命名函数表达式
```js
const factorial = (function f(num) {
	if (num <= 1) {
		return 1;
	} else {
		return num * f(num - 1);
	}
});
```
这里创建了一个命名函数表达式 `f()`(f被称为内部名称)
然后将它赋值给了变量 `factorial`(factorial被称为外部名称)
即使把函数赋值给另一个变量，函数表达式的名称 `f` 也不变，因此递归调用不会有问题
这个模式在严格模式和非严格模式下都可以使用
+ ==函数内部名称(即`f`)只能在函数内部使用==
	函数外部名称(即`factorial`)在内部外不都能使用
	如果在内部使用函数外部名称, 就需要担心 函数逻辑 与 函数外部名 紧密耦合的问题
	具体转到[[9. 函数内部#`arguments`对象|使用callee解耦]]
+ ==此时函数的name属性由内部名称决定==
	转到[[2. 函数名#命名函数表达式的name属性|命名函数表达式的name属性]]
