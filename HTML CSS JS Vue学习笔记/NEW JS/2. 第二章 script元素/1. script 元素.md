# \<script>元素属性
在HTML中插入js的主要方法
共有8个属性
+ async[可选] :  表示应该立即开始下载脚本，但不能阻止其他页面动作
	比如下载资源或等待其他脚本加载 只对外部脚本文件有效 
+ charset[可选]： 使用 src 属性指定的代码字符集 
	这个属性很少使用，因为大多数浏览器不在乎它的值 
+ crossorigin : 配置相关请求的 *CORS(跨源资源共享)* 设置
	crossorigin="anonymous"配置文件请求不必设置凭据标志 
	crossorigin="use-credentials"设置凭据标志，意味着出站请求会包含凭据 
+ defer[可选] : 表示脚本可以延迟到文档完全被解析和显示之后再执行
	只对外部脚本有效
+ integrity[可选] :  允许比对接收到的资源和指定的加密签名以验证子资源完整性
	（SRI，Subresource Integrity）
	如果接收到的资源的签名与这个属性指定的签名不匹配，则页面会报错，脚本不会执行 
	这个属性可以用于确保内容分发网络（CDN，Content Delivery Network）不会提供恶意内容 
+ src[可选] : 表示包含要执行的代码的外部文件
+ language[废弃] : 
+ type[可选] : 代替language , 表示代码块中 ==脚本语言的内容类型（也称 MIME 类型）==
	关于[MIME 类型](https://www.runoob.com/http/mime-types.html)
## src
`src` 属性的值可以是一个完整的URL
例如:
```js
<script src="http://www.somewhere.com/afile.js"></script>
```
当浏览器解析到这个资源时, 会向该路径发送一个*GET请求(HTTP请求的两种方法之一)*
为确保返回的js的安全, 应配合使用`integrity`属性进行验证
+ 使用了`src`属性的\<script>, 不应再包含js代码, 否则忽略行内代码
+ 按照惯例, 外部JavaScript代码的扩展名(后缀名)是`.js`
	1. 但浏览器不会检查所包含JavaScript文件的扩展名
		这就为使用服务器端脚本语言动态生成 JavaScript 代码[动态调整]，
		或者在浏览器中将 JavaScript扩展语言（如TypeScript，或React的 JSX）
		转译为JavaScript提供了可能性[转译] 
	2.  不过要注意，服务器经常会根据文件扩展来确定响应的正确 MIME 类型 
		如果不打算使用.js 扩展名，一定要确保服务器能返回正确的 MIME 类型 
# 使用
## 嵌套 \ 多个
不能嵌套\<script>标签
可以有多个\<script>标签, 浏览器会依次解释
前提是\<script>没有defer和asyns属性
# 位置
通常将\<script>标签放到\<body>的内容后面
页面会在处理 JavaScript 代码之前完全渲染页面
减少空白页面的持续时间
# 推迟执行脚本
使用defer属性, 脚本会被延迟到==整个页面解析完毕之后==在运行
此时\<script>就可以放到\<body>标签前面
+ 只对外部脚本有效
+ 当多个\<script>有defer属性时, 仍会按照顺序解释
	而且两者都会在 [DOMContentLoaded](https://developer.mozilla.org/zh-CN/docs/Web/API/Document/DOMContentLoaded_event) 事件之前执行
	不过在实际当中
	推迟执行的脚本不一定总会按顺序执行或者在 DOMContentLoaded事件之前执行
	因此最好只包含一个这样的脚本 
# 异步执行脚本(异步脚本)
与`defer`属性类似, `async`也改变了脚本的处理在加载时得顺序
`async`会使外部脚本立刻开始下载
==但多个脚本添加`async`属性时, 脚本之间执行的次序不能保证==
所以多个添加`async`属性的脚本应该之间灭有依赖关系
给脚本添加 async 属性的目的是告诉浏览器
==不必等脚本下载和执行完后再加载页面==，同样==也不必等到该异步脚本下载和执行后再加载其他脚本==
所以它可能会在页面的加载过程中并行执行, 所以异步脚本不应该在加载期间修改 DOM
# 动态加载脚本
通过调用DOM API, 动态添加指定脚本
```js
let script = document.createElement('script');
script.src = 'gibberish.js';
document.head.appendChild(script);
```
以这种方式添加的脚本, ==默认为异步脚本(async)==
但不是所有浏览器都支持`async`属性, 所以在添加脚本时, 将其设置为同步加载
```js
let script = document.createElement('script');
script.src = 'gibberish.js';
script.async = false;
document.head.appendChild(script);
```
%%
以这种方式获取的资源对浏览器预加载器是不可见的 
这会严重影响它们在资源获取队列中的优先级 
根据应用程序的工作方式以及怎么使用，这种方式可能会严重影响性能 
要想让预加载器知道这些动态请求文件的存在，可以在文档头部显式声明它们：
```js
<link rel="preload" href="gibberish.js">
```
