>异步行为是 JavaScript 的基础，但以前的实现不理想
在早期的 JavaScript 中，只支持定义回调函数来表明异步操作完成
串联多个异步操作是一个常见的问题，通常需要深度嵌套的回调函数（俗称“回调地狱”）来解决
假设有以下异步函数，使用了 `setTimeout` 在一秒钟之后执行某些操作：
```js
function double(value) {
	setTimeout(() => setTimeout(console.log, 0, value * 2), 1000);
}
double(3);
// 6（大约 1000 毫秒之后）
```
这里的代码没什么神秘的，但关键是理解为什么说它是一个异步函数
`setTimeout` 可以定义一个在指定时间之后会被调度执行的回调函数
对这个例子而言，1000 毫秒之后，JavaScript 运行时会把回调函数推到自己的消息队列上去等待执行
	转到[[消息队列|消息队列]]
推到队列之后，回调什么时候出列被执行对 JavaScript 代码就完全不可见了
还有一点，double()函数在 setTimeout 成功调度异步操作之后会立即退出
