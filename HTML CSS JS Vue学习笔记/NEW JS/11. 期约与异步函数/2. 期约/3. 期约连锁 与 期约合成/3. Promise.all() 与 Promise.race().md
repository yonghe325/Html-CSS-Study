`Promise` 类提供两个将多个期约实例组合成一个期约的静态方法：
`Promise.all()`和 `Promise.race()`
而 合成后 期约的行为 取决于 ==内部期约== 的行为
# `Promise.all()`方法
`Promise.all()`静态方法创建的期约会 在一组期约全部解决之后再解决
	即所有内部期约
这个静态方法 ==必须== ==接收一个可迭代对象==，返回一个新期约
## 参数
```js
let p1 = Promise.all([
	Promise.resolve(),//内部期约
	Promise.resolve()//内部期约
]);
```
当传入可迭代对象时, 会对所有元素调用`Promise.resolve()`方法转换为 期约
```js
// 可迭代对象中的元素会通过 Promise.resolve()转换为期约
let p2 = Promise.all([3, 4]);
```
传入空的可迭代对象时, `Promise.all()`的行为等价于`Promise.resolve()`
```js
let p3 = Promise.all([]);
//等价于
let p3 = Promise.resolve();
```
当不传入参数时, 抛出异常
```js
let p4 = Promise.all();
// TypeError: cannot read Symbol.iterator of undefined
```
---
## 行为
合成的期约只会在每个包含的期约都解决之后才解决
	否则永久处于`pending`状态
```js
let p = Promise.all([
	Promise.resolve(),
	new Promise((resolve, reject) => setTimeout(resolve, 1000))
]);
setTimeout(console.log, 0, p); // Promise <pending>
p.then(() => setTimeout(console.log, 0, 'all() resolved!'));
// all() resolved!（大约 1 秒后）
```
如果其中一个期约状态为拒绝
则合成的期约落定为拒绝
```js
let p2 = Promise.all([
	Promise.resolve(),
	Promise.reject(),
	Promise.resolve()
]);
setTimeout(console.log, 0, p2); // Promise <rejected>
// Uncaught (in promise) undefined
```
## 返回值
如果所有期约都成功解决，则合成期约的解决==值==就是所有包含期约==解决值的数组==
按照迭代器顺序:
```js
let p = Promise.all([
	Promise.resolve(3),
	Promise.resolve(),
	Promise.resolve(4)
]);

p.then((values) => setTimeout(console.log, 0, values)); // [3, undefined, 4]
```
如果有期约拒绝，则第一个拒绝的期约会将自己的理由作为合成期约的拒绝理由
之后再拒绝的期约不会影响最终期约的拒绝理由(幂等)
不过，这并不影响所有包含期约正常的拒绝操作
合成的期约会==静默处理所有包含期约的拒绝操作==
	这意味着如果有 子期约即使没有错误处理程序
	程序也不会触发`unhandledrejection`事件
如下所示
```js
// 虽然只有第一个期约的拒绝理由会进入
// 拒绝处理程序，第二个期约的拒绝也
// 会被静默处理，不会有错误跑掉
let p = Promise.all([
	Promise.reject(3),
	new Promise((resolve, reject) => setTimeout(reject, 1000))
]);
p.catch((reason) => setTimeout(console.log, 0, reason)); // 3
// 没有未处理的错误
```
# `Promise.race()`方法
`Promise.race()`静态方法返回一个包装期约
是一组集合中最先解决或拒绝的期约的镜像
## 参数
这个方法接收一个可迭代对象，返回一个新期约：
```js
let p1 = Promise.race([
	Promise.resolve(),
	Promise.resolve()
]);
```
关于传入参数的限制, 与`Promise.all()`相同
	转到[[3. Promise.all() 与 Promise.race()#参数|Promise.all()的参数限制]]
## 行为
`Promise.race()`不会对解决或拒绝的期约区别对待
无论是解决还是拒绝，只要是==第一个落定的期约==
`Promise.race()`就会包装其解决值或拒绝理由并返回新期约
1. 解决先发生，超时后的拒绝被忽略
```js
let p1 = Promise.race([
	Promise.resolve(3),
	new Promise((resolve, reject) => setTimeout(reject, 1000))
]);
setTimeout(console.log, 0, p1); // Promise <resolved>: 3
```
2. 拒绝先发生，超时后的解决被忽略
```js
let p2 = Promise.race([
	Promise.reject(4),
	new Promise((resolve, reject) => setTimeout(resolve, 1000))
]);
setTimeout(console.log, 0, p2); // Promise <rejected>: 4
```
3. 当同时发生时(在同步操作或异步操作中), 迭代顺序决定了落定顺序
```js
let p3 = Promise.race([
	Promise.resolve(5),
	Promise.resolve(6),
	Promise.resolve(7)
]);
setTimeout(console.log, 0, p3); // Promise <resolved>: 5
//同步

let p3 = Promise.race([
  new Promise((resolve,reject)=>setTimeout(resolve(1),1000)),
  new Promise((resolve,reject)=>setTimeout(resolve(2),1000)),
  new Promise((resolve,reject)=>setTimeout(resolve(3),1000)),
  new Promise((resolve,reject)=>setTimeout(resolve(4),1000))
]);
setTimeout(console.log, 0, p3); // Promise <resolved>: 1
//异步
```
4. 静默处理所有包含期约的拒绝操作, 与`Promise.all()`相同
```js
// 虽然只有第一个期约的拒绝理由会进入
// 拒绝处理程序，第二个期约的拒绝也
// 会被静默处理，不会有错误跑掉
let p = Promise.race([
	Promise.reject(3),
	new Promise((resolve, reject) => setTimeout(reject, 1000))
]);
p.catch((reason) => setTimeout(console.log, 0, reason)); // 3
// 没有未处理的错误
```
