# 期约状态机
期约是一个有状态的对象，可能处于如下 3 种状态之一:
+ **待定**(pending)
	pending状态是期约最初始的状态
	在pending状态下, ==期约仅可以转变一次状态==
	且转变之后不能在改变状态
+ **兑现**(fulfilled / resolve)
	处于pending状态的期约
	可以**落定**为代表 成功状态 的 兑现(resolve)状态
+ **拒绝**(reject)
	处于pending状态的期约
	可以**落定**为代表 失败状态 的 拒绝(resolve)状态
---
# 期约的特性
+ 期约的状态是私有的
	所以只能在期约内部通过特定方法改变期约状态
		转到[[3. 通过执行函数控制期约状态|resolve() 与 reject()]]
	不能直接通过 JavaScript 检测到
	这主要是为了避免根据读取到的期约状态
	以同步方式处理期约对象
+ 期约的状态也不能被外部 JavaScript 代码修改
	这与不能读取该状态的原因是一样的：
	期约故意将异步行为封装起来
	从而隔离外部的同步代码
+ 状态为 **拒绝**(reject) 的期约必须被`catch`捕获

	否则程序会终止, 抛出警告
