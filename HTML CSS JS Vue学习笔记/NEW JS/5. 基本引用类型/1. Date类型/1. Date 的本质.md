Date类型将日期保存为自协调世界时（UTC，Universal Time Coordinated）时间 
1970 年 1 月 1 日午夜（零时）至今所经过的毫秒数(时间戳)
使用这种格式, 精确表示1970 年 1 月 1 日之前及之后 285 616 年的所有日期时间

通过调用Date构造函数来创建日期对象
```js
let now = new Date()
```
# Date构造函数 及其 Date对象的方法
在不给Date构造函数, Date构造函数将基于当前时间创建Date对象
如果创建其他时间的date对象, 则必须传入以 协调世界时为基准的 ==一个毫秒数==
可以通过 `Date.parse()` 和` Date.UTC()` 两个辅助方法, 将日期转化为对象的毫秒数
还有`Date.now()`方法直接返回当前的时间戳
+ `Date.parse()`
	接受一个 符合规定格式的 表示日期的 字符串参数
	所有格式如下 : 
	+ `"mm/dd/yyyy"` : `"5/23/2.19"` 
	+ `月名 dd, yyyy` : `"May 23, 2019"`
	+ `"周几 月名 日 年 时:分:秒 时区"` : `""Tue May 23 2019 00:00:00 GMT-0700"`
	+ `“YYYY-MM-DDTHH:mm:ss.sssZ` : `"2019-05-23T00:00:00"`
	如果不符合上述格式, `Date.parse()`将返回`NaN`
	如果直接将符合格式的字符串传给Date构造函数
	Date构造函数会在内部先调用`Date.parse()`方法, 然后在创建符合的日期对象
	```js
	let someDate = new Date(Date.parse("May 23, 2019"))
	//等价于
	let someDate = new Date("May 23, 2019")
	```
	如果格式正确, 但日期不符合逻辑(越界), 不同浏览器有不同处理方式
	可能会直接返回当前时间, 
+ `Date.UTC()`
	`Date.UTC()`返回值与`Date.parse()`相同 , 都是返回特定的时间毫秒数, 只是传参不同
	`Date.UTC()`的传参只有一种格式
	`年 零起点月数(1月是0) 日(1~31) 时(0~23) 分 秒 毫秒`
	```js
	// GMT 时间 2000 年 1 月 1 日零点
	let y2k = new Date(Date.UTC(2000, 0));
	// GMT 时间 2005 年 5 月 5 日下午 5 点 55 分 55 秒
	let allFives = new Date(Date.UTC(2005, 4, 5, 17, 55, 55));
	```
	只有年月是必需的, 日默认为1, 其他默认为0
	`Date.UTC()`方法也会被Date构造函数隐式使用
+ `Date.now()`
	`Date.now()`方法会直接返回当前的时间戳
	以实现计算时间之差
	```js
	// 起始时间
	let start = Date.now();
	// 调用函数
	doSomething();
	// 结束时间
	let stop = Date.now(),
	result = stop - start;
	```
