创建完范围之后, 浏览器会在内部创建一个文档片段节点
	即一个`DocumentFragment`对象
用于包含范围选区中的节点
==选取内的内容必须格式完好==
## 补全DOM结构
如果一个选取的起点和终点都在一个文本节点内部
那选区内的格式就不是一个完好的DOM结构
不过，范围能够确定缺失的开始和结束标签
从而可以==补全出有效的 DOM 结构==, 以便后续操作
例如:
```js
<p id="p1"><b>Hello</b> world!</p>

let range = document.createRange();
range.setStart(helloNode, 2);
range.setEnd(worldNode, 3);
```
选中了 `llo</b> wo`
但不是完整的DOM结构
因此会在后台动态补上对应的标签, 如下
```js
<p><b>He</b><b>llo</b> world!</p>
```
`llo`会被动态的补上`<b>`标签, 而` world!`则会被拆分成两个文本节点, 其中一个就是` wo`

---
补全可以发生在原DOM中, 也可以发生在选取的片段中
如果是一个`DocumentFragment`对象, 则片段会放在一个**抽象的、没有任何标签的虚拟容器**, 可以看成Node类型的一种
```js
<div><b>llo</b> wo</div>
```
# `deleteContents()`方法 -> 删除范围
这个方法会从文档中删除范围包含的节点
```js
<p id="p1"><b>Hello</b> world!</p>

let p1 = document.getElementById("p1"),
helloNode = p1.firstChild.firstChild,
worldNode = p1.lastChild,
range = document.createRange();
range.setStart(helloNode, 2);
range.setEnd(worldNode, 3);

range.deleteContents();
```
执行上面的代码之后，页面中的 HTML 会变成这样：
```js
<p><b>He</b>rld!</p>
```
此时开始与结束节点不完整的DOM结构会被修改, 使之变得完整
# `extractContents()`方法 -> 删除原范围, 返回片段
与`deleteContents()`方法类似, `extractContents()`方法同样会删除原DOM中的范围
但会返回范围对应的文档片段(一个`DocumentFragment`对象) 
这样就可以把选取的片段插入到文档的其他地方
```js
let p1 = document.getElementById("p1"),
helloNode = p1.firstChild.firstChild,
worldNode = p1.lastChild,
range = document.createRange();

range.setStart(helloNode, 2);
range.setEnd(worldNode, 3);

let fragment = range.extractContents();
	// 返回文档片段
p1.parentNode.appendChild(fragment);
	// 插入回文档
```
得到以下HTML
```js
<p><b>He</b>rld!</p>
<b>llo</b> wo
```
# `cloneContents()`方法 -> 创建副本
使用 `cloneContents()`会对范围创建一个副本, 返回一个`DocumentFragment`对象
而不会对原文档进行任何操作
```js
let p1 = document.getElementById("p1"),
helloNode = p1.firstChild.firstChild,
worldNode = p1.lastChild,
range = document.createRange();
range.setStart(helloNode, 2);
range.setEnd(worldNode, 3);
let fragment = range.cloneContents();
p1.parentNode.appendChild(fragment);
```
这个方法跟 `extractContents()`很相似，因为它们都返回文档片段
	即`extractContents()`返回的`FragmentDocument`对象就是原文档中的范围
主要区别是 `cloneContents()` 返回的文档片段包含范围中节点的副本，而非实际的节点
	即`cloneContents()`返回的`FragmentDocument`是范围在内存中的复制
