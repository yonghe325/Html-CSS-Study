# DocumentType的变化
转到[[0. DocumentType类型|DocumentType类型]]
DocumentType 新增了 3 个属性：
+ `publicId` : 公共标识符
+ `systemId` : 系统标识符, 转到[[1. DTD|DTD]]
+ `internalSubset` : 内部子集, 里面可以定义一些当前文档专用的实体或符号
	类似于宏定义, ==此语法在HTML文档中实现==
		转到[[1. DTD#HTML5中的DTD|HTML5中的DTD]]
	HTML有其他实现类似功能的方式
# `publicId`、`systemId` 
`publicId`、`systemId` 属性表示文档类型声明中==有效但无法使用 DOM1 API 访问的数据==
比如下面这个 HTML 文档类型声明：
```js
<!DOCTYPE HTML PUBLIC "-// W3C// DTD HTML 4.01// EN" "http://www.w3.org/TR/html4/strict.dtd">
```
其中 publicId 是"-// W3C// DTD HTML 4.01// EN"
而 systemId 是"http\://www.w3.org/TR/html4/strict.dtd"
通常在网页中很少需要访问这些信息
# `internalSubset` 
`internalSubset` 用于访问文档类型声明中可能包含的额外定义，如下面的例子所示
```js
<!DOCTYPE html PUBLIC 
	"-// W3C// DTD XHTML 1.0 Strict// EN" 
	"http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd"
	[<!ELEMENT name (#PCDATA)>] >
```
对于以上声明，`document.doctype.internalSubse`t 会返回"`<!ELEMENT name (#PCDATA)>`"。
HTML 文档中几乎不会涉及文档类型的内部子集，XML 文档中稍微常用一些
