# importNode()
Document 类型的更新中唯一跟命名空间无关的方法是 `importNode()`
这个方法的目的是从其他文档获取一个节点并导入到新文档，以便将其插入新文档
每个节点都有一个 ownerDocument 属性，表示所属文档
	转到[[0. Document类型|ownerDocument属性]]
如果调用 `appendChild()`方法时传入节点的 `ownerDocument` 不是指向当前文档, 则会发生错误
而调用 `importNode()`导入其他文档的节点会返回一个新节点
这个新节点的 `ownerDocument` 属性是正确的

---
`importNode()`方法跟 [[4. 其他方法#`cloneNode()`方法|cloneNode()]] 方法类似
1. 接收两个参数：
	+ 要复制的节点
	+ 表示是否同时复制子树的布尔值(是否深复制)
2. 返回结果是适合在当前文档中使用的新节点

```js
let newNode = document.importNode(oldNode, true); // 导入节点及所有后代
document.body.appendChild(newNode);
```
这个方法在 HTML 中使用得并不多，在 XML 文档中的使用会更多一些（第 22 章会深入讨论）
# defaultView 属性
属于DOM2 View模块
是一个指向拥有当前文档的窗口(或窗格\<frame>)的指针
这个规范中并没有明确视图何时可用，因此这是添加的唯一一个属性
defaultView 属性得到了除 IE8 及更早版本之外所有浏览器的支持
IE8 及更早版本支持等价的 parentWindow 属性，Opera 也支持这个属性
因此要确定拥有文档的窗口，可以使用以下代码(客户端检测)
```js
let parentWindow = document.defaultView || document.parentWindow;
```
# `document.implementation` 对象
`document.implementation` 对象类似于创建新文档的工厂
DOM2 Core为其添加了两个新的方法, 用以创建新的 DocumentType对象 与 新文档
至于新文档如何打开, 转到[[1. 打开新文档|打开新文档]]
## createDocumentType()
+ 该方法接受三个参数:
	+ 文档类型名称
	+ publicId
	+ systemId
+ 返回一个新的HTML文档对象
```js
let doctype = document.implementation.createDocumentType("html",// 文档类型名称
	"-// W3C// DTD HTML 4.01// EN", // publicID
	"http://www.w3.org/TR/html4/strict.dtd" // systemId
	);
```
==已有文档的文档类型不可更改，因此 createDocumentType()只在创建新文档时才会用到==
而创建新 XML 文档要使用 createDocument()方法
## createDocument()
+ 该方法同样接受三个参数:
	+ 文档元素的命名空间(namespaceURL)
	+ 文档元素的标签名
	+ 文档类型
+ 返回一个新的空文档
```js
let doc = document.implementation.createDocument("", "root", null);
// 返回一个空的XML文档, 仅指定 <root> 元素为文档的元素

let doctype = document.implementation.createDocumentType("html",
	"-// W3C// DTD XHTML 1.0 Strict// EN",
	"http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd");
	let doc = document.implementation.createDocument("http://www.w3.org/1999/xhtml","html", doctype);
// 结合createDocumentType()方法
// 可以创建一个新的XHTML文档, 文档内部仅有一个<html>元素
```
## createHTMLDocument()
与`createDocument()`方法类似, 该方法用以创建新的HTML文档(一个 `HTMLDocument` 类型的实例)
且新文档包含完整的文档结构, 即包含\<html>, \<head>,\<title>,\<body>等元素
不同的是, `createHTMLDocument()`方法仅接受一个参数, 即新文档的标题(放入\<title>元素中)`
```js
let htmldoc = document.implementation.createHTMLDocument("New Doc");
console.log(htmldoc.title); // "New Doc"
console.log(typeof htmldoc.body); // "object"
```
如果想要在新文档中插入新元素
则需要再源文档中创建新元素, 并将其插入到新文档中
可以使用`appendChild()`方法, 新元素的`owerDocument`属性会自动迁移到新文档
带这种迁移不稳定, 最好使用`importNode()`方法, 直接在新文档中复制新元素
