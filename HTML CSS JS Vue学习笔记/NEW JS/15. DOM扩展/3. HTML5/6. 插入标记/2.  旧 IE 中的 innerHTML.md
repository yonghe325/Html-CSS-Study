# 通过 innerHTML 插入 script 元素
在所有现代浏览器中，通过 innerHTML 插入的\<script>标签是不会执行的
而在 IE8 及之前的版本中，只要这样插入的\<script>元素指定了 defer 属性
且\<script>之前是“受控元素”（scopedelement），那就是可以执行的
\<script>元素与\<style>或注释一样，都是“非受控元素”（NoScopeelement）
也就是在页面上看不到它们
IE 会把 innerHTML 中从非受控元素开始的内容都删掉，也就是说下面的例子是行不通的：
```js
div.innerHTML = "<script defer>console.log('hi');<\/script>";
// 行不通
```
`innerHTML`字符串会在第一个非受控元素开始, 清空`innerHTML`字符串
所以只需要在\<script defer>前加上一个受控元素
例如文本节点或没有结束标签的元素（如\<input>）
就可以正常执行
```js
// 以下都可行
div.innerHTML = "_<script defer>console.log('hi');<\/script>";
// _ 可能会影响页面布局, 所以后续可能会删掉
div.innerHTML = "<div> </div><script defer>console.log('hi');<\/script>";
// 空<div>是不行的，必须包含一点内容 , 所以后续也可能会删掉
div.innerHTML = "<input type=\"hidden\"><script defer>console.log('hi');<\/script>";
// <input type=\"hidden\">元素不会影响页面布局, 是最理想的受控元素
```
# 通过 innerHTML 插入 style 元素
通过 innerHTML 插入\<style>也会有类似的问题
多数浏览器支持使用 innerHTML 插入\<style>元素：
```js
div.innerHTML = "<style type=\"text/css\">body {background-color: red; }</style>";
```
但在 IE8 及之前的版本中，\<style>也被认为是非受控元素，所以必须前置一个受控元素：
```js
div.innerHTML = "_<style type=\"text/css\">body {background-color: red; }</style>";
div.removeChild(div.firstChild);
```
