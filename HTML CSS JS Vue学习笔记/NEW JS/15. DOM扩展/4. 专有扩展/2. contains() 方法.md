DOM 编程中经常需要确定一个元素是不是另一个元素的后代
IE 首先引入了 contains()方法，让开发者可以在不遍历 DOM 的情况下获取这个信息
contains()方法应该==在要搜索的祖先元素上调用==，==参数是待确定的目标节点==
如果目标节点是被搜索节点的后代，contains()返回 true，否则返回 false
```js
console.log(document.documentElement.contains(document.body)); // true
```
# compareDocumentPosition()方法
这是在DOM3 中的方法
同样可以确定节点之间的关系
返回对应关系的掩码
1. 0x1(1) : 断开（传入的节点不在文档中）
2. 0x2(2) : 领先（传入的节点在 DOM 树中位于参考节点之前）
3. 0x4(4) : 随后（传入的节点在 DOM 树中位于参考节点之后）
4. 0x8(8) : 包含（传入的节点是参考节点的祖先）
5. 0x10(16) : 被包含（传入的节点是参考节点的后代）

---
`contains()`方法可以直接模仿`compareDocumentPosition()`返回 掩码0x4 与 掩码0x10 的方法
```js
let result = document.documentElement.compareDocumentPosition(document.body);
console.log(!!(result & 0x10));
// result的值为20 , 即0x20
// 代表 0x4 与 0x10
// 对result 和 0x10 应用按位与会返回非零值
// 而两个叹号将这个值转换成对应的布尔值
```
