# 组件
一共分为三个组件
1. Model: 数据层
	变量, 数据等都存储在这里
2. View: 
	用户看到的视图, Html在这里
3. ViewModel
	Model和View的桥梁
	自动监听 Model 变化并更新 View
	同时捕获 View 事件并更新 Model
	即双向绑定
# 核心
其核心是==数据劫持==+==观察者模式==
## 在Vue中的实现方式
### 数据劫持
Vue 3 使用 `Proxy` 来拦截对数据的操作
+ `Proxy`是JavaScript中的==拦截器==
	当被监听的对象被读取或被修改时
	插入新的代码逻辑
使用`Proxy`才可以实现Vue的响应式系统
### 视图更新
当数据变化时, 为实现双向绑定的功能
Vue 的**渲染函数**会生成**虚拟 DOM**
#### 渲染函数 
接收数据, 返回描述界面长什么样的JS对象(虚拟DOM)
在Vue中使用模板方式书写HTML(模板)
+ 模板方式
	模板方式就是**用类似 HTML 的语法来描述界面**，是 Vue 推荐的默认写法
	`.vue` 文件里写的 `<template>` 部分，就是模板
Vue最终会把模板==编译成渲染函数==
```js
<template>
  <div class="container">
    <h1>{{ title }}</h1>
    <button @click="handleClick">点我</button>
  </div>
</template>
//上述代码编译成渲染函数后如下
//（简化版）
function render() {
  return h('div', { class: 'container' }, [
    h('h1', {}, this.title),
    h('button', { onClick: this.handleClick }, '点我')
  ]);
}
```
这个 `h()` 函数（也叫 **hyperscript** 或 **createVNode**）执行后会返回一个**虚拟 DOM 节点**：
```js
// 虚拟 DOM 对象（简化版）
{
  type: 'div',
  props: { class: 'container' },
  children: [
    { type: 'h1', props: {}, children: 'Hello' },
    { type: 'button', props: { onClick: fn }, children: '点我' }
  ]
}
```
##### 渲染函数的执行流程
1. **数据变化**：`this.title` 从 `'Hello'` 变成 `'World'`
2. **渲染函数重新执行**：`render()` 再次被调用
3. **生成新虚拟 DOM**：产生一个新的对象树
4. **对比差异**：Vue 比较新旧两棵虚拟 DOM 树
5. **更新真实 DOM**：只把变化的部分应用到页面上
