一个用于构建**用户界面**的**渐进式 JavaScript 框架**
+ 构建用户界面:
	Vue 的核心任务就是**把数据变成页面**，并且让数据和页面保持同步
	这也是MVVM框架的核心
+ 渐进式框架
	Vue可以应对不同使用场景

| 使用程度     | 怎么做                                 | 适用场景       |
| -------- | ----------------------------------- | ---------- |
| **轻量使用** | 用 `<script>` 引入，在现有页面里加一点交互         | 老项目优化、简单页面 |
| **中等使用** | 用单文件组件（`.vue`），配合构建工具               | 完整的前端应用    |
| **重度使用** | 全家桶：Vue + Vue Router + Pinia + Vite | 大型企业级项目    |
# Vue的核心特点
## 声明式渲染
选对于原生JavaScript这种命令式渲染
使用Vue只需要直接修改需要的内容
不像原生JavaScript那样需要先捕获再修改等
那样一步一步等告诉浏览器如何修改
```js
<!-- 声明式：我只管说想要什么 -->
<template>
  <div>{{ message }}</div>
</template>

<!-- 对比命令式（原生JS）：我得告诉浏览器每一步 -->
<div id="app"></div>
<script>
  const div = document.getElementById('app');
  div.innerText = message;  // 手动操作
</script>
```
## 响应式系统
数据变了, 视图也会跟着变
```js
// 你只管改数据
data() {
  return {
    count: 0
  }
},
methods: {
  increment() {
    this.count++;  // 页面上的 {{ count }} 自动变
  }
}
```
## 组件化
把页面拆成一个个**独立可复用的组件**
```js
App.vue（整个页面）
├─ Header.vue（头部）
├─ Sidebar.vue（侧边栏）
├─ UserList.vue（用户列表）
│  ├─ UserItem.vue（每个用户项）
│  └─ UserDetail.vue（用户详情）
└─ Footer.vue（底部）
```
每个组件都是独立的
