与c++,c, java类似
Number 类型也使用 IEEE 754 格式表示整数和浮点数
所以和其他语言类似, 也会有 `+0` 和 `-0`, JavaScript对此不做区分
只不过JavaScript使用不同数值的字面量区分整形, 浮点型
## 整形
+ 十进制整数
	直接书写
+ 八进制字面量
	第一个数字必须为零, 且符合八进制的进制规律
	否则将忽略第一个零, 将剩余数字转换为十进制数字
	八进制字面量 在严格模式下无效, 会抛出异常
+ 十六进制字面量
	必须以0x开头, 只能是小写x
	十六进制中的字母不区分大小写 
## 浮点型
在不使用科学计数法时, 浮点型字面量必须包含小数点
当小数点前的整数为0时, 可以省略
当小数点后没有数字时, 被认为时整数
因为使用 IEEE 754 格式
所以在进行浮点数运算时, 常有精准度问题
此时就应该划定可接受范围
```js
//在JavaScript等使用 IEEE 754 格式的语言中
0.1+0.2 = 0.300 000 000 000 000 04
//所以
if (a + b == 0.3) { 
	console.log("You got 0.3.");
}
//有隐患
//当 a=0.1 b = 0.2时, 不会执行
```
# 值的范围
与其他语言类似, JavaScript里同样有内置的最小值
但没有单独的整形, 浮点型最小值
二者共用最小值, 最大值
```js
Number.MIN_VALUE = 5e-324
//最小值
Number.MAX_VALUE = 1.797 693 134 862 315 7e+308
//最大值

//二者计算将返回Infinity
```
当值的范围查过最大值, 最小值时
会自动转化为`+Infinity` 或 `-Infinity`
==该变量将不再能用于任何计算==
可以使用内置函数判断该变量的值是否为`Infinity`
```js
isFinite(<变量>)
//是infinite, 返回 true
//否则返回false
```
# NaN 
NaN(Not a Number)
当返回数值的操作失败时, 将返回NaN
```js
console.log(0/0); // NaN
console.log(-0/+0); // NaN
console.log(5/0); // Infinity
console.log(5/-0); // -Infinity
```
任何实际NaN的运算操作, 始终返回NaN
NaN不等于 包括NaN在内的任何值
所以
```js
console.log(NaN == NaN); // false
```
为此, 应使用`isNaN()`函数判断是否为`NaN`
`isNaN()`会先潘队该值是否能被转换为`Number`类型
可以被转换, 返回`true`, 否则返回`false`
```js
console.log(isNaN(NaN)); // true
console.log(isNaN(10)); // false，10 是数值
console.log(isNaN("10")); // false，可以转换为数值 10
console.log(isNaN("blue")); // true，不可以转换为数值
console.log(isNaN(true)); // false，可以转换为数值 1
```
# 数值转换
有三个函数可以将非数值型转换为数值
`Number()`, `parseInt()`, `parseFloat`
## `Number()` 可用于所有类型
+ 将所有可以转换为`Boolean`值`false`的值, 转换为 0, `true`转换为0
+ `Undefined`转换为`NaN`
+ 字符串 : 
	+ 如果字符串中包含数值字符, 包括数字字符前面有`+`,`-`
		则转换为一个十进制数, 忽略前面的所有 0
	+ 如果字符中包含正确浮点型格式的的数值,如"1.1"
		则会转换为浮点值, 忽略前面的所有 0
	+ 如果字符串包含有效的十六进制格式如"0xf"
		则会转换为与该十六进制值对应的十进制整数值 
	+  如果是空字符串, 不包含字符
		则返回 0
	+  如果字符串包含除上述情况之外的其他字符
		则返回 NaN
+ 对象 : 先调用`valueOf()`方法, 并按照上述方法转换
	如果是NaN, 则调用`toString()`方法,在按照字符串的规则转化
## `parseInt()`
1. 字符串最前面的空格会被忽略
2. 从第一个非空格字符开始转换
	如果第一个字符不是数值字符, 加号或减号,
	parseInt()立即返回 NaN
	这意味着空字符串也会返回 NaN (这一点与 Number()不一样，它返回 0)
3. 如果第一个字符是数值字符, 加号或减号, 
	则继续依次检测每个字符, 直到字符串末尾, 或碰到非数值字符
```js
let num1 = parseInt("1234blue"); // 1234
let num2 = parseInt(""); // NaN
let num3 = parseInt("0xA"); // 10，解释为十六进制整数
let num4 = parseInt(22.5); // 22
let num5 = parseInt("70"); // 70，解释为十进制值
let num6 = parseInt("0xf"); // 15，解释为十六进制整数
```
`parseInt()`可传入第二个参数, 用来制定将字符串转换为何种进制
```js
let num = parseInt("0xAF", 16); // 175
//如果提供了十六进制参数，那么字符串前面的"0x"可以省掉：
let num1 = parseInt("AF", 16); // 175
//没有第二个参数, 则不会识别
let num2 = parseInt("AF"); // NaN
	//////////////////////////////////////////////////////////////////////
let num1 = parseInt("10", 2); // 2，按二进制解析
let num2 = parseInt("10", 8); // 8，按八进制解析
let num3 = parseInt("10", 10); // 10，按十进制解析
let num4 = parseInt("10", 16); // 16，按十六进制解析
```
## `parseFloat`
与`parseInt`的识别规律类似, 但不同的是
+ 只会识别第一个出现的` . `, 且识别截止到第二个`.`
```js
22.34.5 ==> 22.34
```
+ 它始终忽略字符串开头的零, 且只解析十进制数值
```js
let num1 = parseFloat("1234blue"); // 1234，按整数解析
let num2 = parseFloat("0xA"); // 0
let num3 = parseFloat("22.5"); // 22.5
let num4 = parseFloat("22.34.5"); // 22.34
let num5 = parseFloat("0908.5"); // 908.5
let num6 = parseFloat("3.125e7"); // 31250000
```
