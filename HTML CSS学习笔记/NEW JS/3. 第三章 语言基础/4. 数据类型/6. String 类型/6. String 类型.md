# 字符字面量
与其他语言类似, JavaScript同样包含各种转义字符

|   字面量    | 含义                              |
| :------: | :------------------------------ |
|   `\n`   | 换行                              |
|   `\t`   | 制表符                             |
|   `\b`   | 退格                              |
|   `\r`   | 回车                              |
|   `\f`   | 换页                              |
|   `\\`   | 反斜杠( \\ )                       |
|   `\'`   | 单单引号( ' ), 防止解释该引号              |
|   `\"`   | 单双引号( " )                       |
|   `\`    | 单反引号( \` )                      |
|  `\xnn`  | 以十六进制编码 nn 表示的 ==ASCLL== 字符     |
| `\unnnn` | 以十六进制编码 nnnn 表示的 ==Unicode== 字符 |
所有字符字面量均只占一个字符长度
```js
let text = "This is the letter sigma: \u03a3.";
console.log(text.length); // 28
// \u03a3 字符算一个长度
```
对于双字节字符, `length`属性返回的值不一定准确
# 字符串的特点
ECMAScript 中的字符串是不可变的（immutable）
即所有字符串一旦创建, 只就不能再被修改
只能重新创建然后修改
```js
let lang = "Java";
lang = lang + "Script";
```
# 转换为字符串

## `toString()`方法
+ `toString()`方法可用于数值, 布尔值, 对象和字符串值
	对字符串值使用, 只会返回本身的副本
	`Null` 和 `Undefined`类型 没有`toString()`方法
+ `toString()` 可以传入一个参数, 作为数值类型转换为字符串类型的 底数, 即进制
	```js
	let num = 10;
	console.log(num.toString()); // "10"
	console.log(num.toString(2)); // "1010"
	console.log(num.toString(8)); // "12"
	console.log(num.toString(10)); // "10"
	console.log(num.toString(16)); // "a"
	```
## `String()`函数
可用于所有类型
+ 对于有`toString()`方法的类型, 等效于直接调用`toString()`方法
	不传如参数
+ 对于没有`toString()`方法的类型, 如`Null` 和`Undefined`类型
	将分别直接转化为 `Null` 和 `Undefined` 的 字面量字符串
## "" + <值>
用加号操作符给一个值加上一个空字符串""也可以将其转换为字符串

# 字符串插值
字符串插值通过在模板字面量里插入${}
并在其中使用一个 JavaScript 表达式实现
```js
let value = 5;
let exponent = 'second';
// 以前，字符串插值是这样实现的：
let interpolatedString =
value + ' to the ' + exponent + ' power is ' + (value * value);
// 现在，可以用模板字面量这样实现：
let interpolatedTemplateLiteral =
`${ value } to the ${ exponent } power is ${ value * value }`;

console.log(interpolatedString); // 5 to the second power is 25
console.log(interpolatedTemplateLiteral); // 5 to the second power is 25
```
所有插入的值都会使用 toString()强制转型为字符串
```js
// 嵌套的模板字符串无须转义：
console.log(`Hello, ${ `World` }!`); // Hello, World!

// 将表达式转换为字符串时会调用 toString()：
let foo = { toString: () => 'World' };
console.log(`Hello, ${ foo }!`); // Hello, World!
	//这里将对象转换为字符串

// 在插值表达式中可以调用函数和方法：
function capitalize(word) {
	return `${ word[0].toUpperCase() }${ word.slice(1) }`;
}
console.log(`${ capitalize('hello') }, ${ capitalize('world') }!`); // Hello, World!

// 此外，模板也可以插入自己之前的值：
let value = '';
function append() {
	value = `${value}abc`
	console.log(value);
}
append(); // abc
append(); // abcabc
append(); // abcabcabc
```
# 模板字面量标签函数
即定标签函数(tag function), 通过标签函数可以自定义插值行为
```js
let a = 6;
let b = 9;
function simpleTag(strings, aValExpression, bValExpression, sumExpression) {
	console.log(strings);
	console.log(aValExpression);
	console.log(bValExpression);
	console.log(sumExpression);
	return 'foobar';
}
let untaggedResult = `${ a } + ${ b } = ${ a + b }`;
let taggedResult = simpleTag`${ a } + ${ b } = ${ a + b }`;
// ["", " + ", " = ", ""]
// 6
// 9
// 15
console.log(untaggedResult); // "6 + 9 = 15"
console.log(taggedResult); // "foobar"
```
简单来说就是对字符串进行处理
对于一个模板字符串函数
1. 第一个参数作为==模板字符串数组==
	该数组为传入的字符串
	如在`${ a } + ${ b } = ${ a + b }`中
	第一个参数`strings = ["" , " + " , " = " , "" ] 
2. 剩余的参数作为字符串的插值, 保存在剩余参数中
	+ ==当确定插值的个数时==, 可以直接在形参中声明插值个数对应的个数的形参
		如在`function simpleTag(strings, aValExpression, bValExpression, sumExpression) {}`中
		`aValExpression, bValExpression, sumExpression`分别代表插值`a, b , a+b`
	+ ==当不确定插值的个数时==, 可以使用==**剩余参数语法**==
		如`function simpleTag(strings, ...value) {}`
		此时所有==传入的插值被放入`value`数组中==
3. 当插值作为模板字符串开头或结尾时, `strings`数组会将第一个值前, 或最后一个值后
	添加一个空字符串`""`, 以确保所有插值均在模板字符串表内
4. 所以在模板字面量标签函数中, 
	模板字符串数组 的元素个数  总比 插值数值的元素个数 ==多一个==
	即使是空插值, 也会用`""`代替
	即
	==对于有 n 个插值的模板字面量
	传给标签函数的表达式参数的个数始终是 n
	而传给标签函数的第一个参数所包含的字符串个数则始终是 n+1==
5. 模板字面量标签函数可以返回任意值, 不一定局限于字符串
# 原始字符串
模板字面量可以直接获取原始的模板字面量内容, 而不是其转义后的内容
例如 换行符 , Unicode字符
可以直接使用默认的`String.raw`标签函数
```js
// Unicode 示例
// \u00A9 是版权符号
console.log(`\u00A9`); // ©
console.log(String.raw`\u00A9`); // \u00A9

// 换行符示例
console.log(`first line\nsecond line`);
// first line
// second line
console.log(String.raw`first line\nsecond line`); 
// "first line\nsecond line"
```
特别的是, `String.raw`标签函数无法转换 实际的隐式换行符
```js
// 对实际的换行符来说是不行的
// 它们不会被转换成转义序列的形式
console.log(`first line
second line`);
// first line
// second line

console.log(String.raw`first line
second line`);
// first line
// second line
```
在标签函数中
对第一个参数, 使用模板字符串数组的`raw`属性, 即可获得每个字符串的原始内容
```js
function printRaw(strings) {
	console.log('Actual characters:');
	for (const string of strings) {
		console.log(string);
	}
	//
	console.log('Escaped characters;');
	for (const rawString of strings.raw) {
		console.log(rawString);
	}
	//原始字符串
}
printRaw`\u00A9${ 'and' }\n`;
// Actual characters:
// ©
//（换行符）
// Escaped characters:
// \u00A9
// \n
```
对于`raw`属性来说, ==只有模板字符串数组有==
因为模板字符串数组是一个特殊的对象, 在传入标签函数中自动创建
