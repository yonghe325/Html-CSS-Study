+ 声明一个 可重新赋值 的 块级作用域 变量
	即申明之后, 只能在该语句块内使用
+ 多个变量声明, 用 ` , ` 连接 , 
	在同一个块作用内, 多个同名变量不能被重复声明
+ 用let声明的变量, 只能在执行到声明所在的位置之后才能被访问
	所以 let 被认为是 **==非提升==** 的
+ 若变量在作用域内没有被重新赋值, 就应该使用 `const`
## for循环中的`let`
在`let`出现之前, 使用`var`声明的迭代变量会渗透到循环体外部
```js
for (var i = 0; i < 5; ++i) {
// 循环逻辑
}
console.log(i); // 5
```
而使用`let`声明迭代变量, 迭代变量只会在块作用域内存活
```js
for (let i = 0; i < 5; ++i) {
// 循环逻辑
}
console.log(i); // ReferenceError
```
## 暂时性死区
+ 使用 let const class 声明的变量
	可以称其从代码块的开始 一直到代码执行到变量声明的位置并被初始化前
	都处于一个“暂时性死区”（Temporal dead zone，TDZ）
	在死区内使用该变量, 会抛出 `ReferenceError` 错误
+ 使用 var声明的变量不同, 当在死区内调用时, 会返回 `undefined`, ==不会抛出异常==
	这与 var 的 **==提升==** 有关
## 全局申明
使用`var`在==全局作用域声明的变量==将会成为==window对象的属性==
可以直接使用window调用该变量
而使用`let`在 全局作用域声明的变量 则不会成为window对象的属性
## 作用域
```js
function(){
	if (true) {
		var name = 'Matt';
		console.log(name); // Matt
	}
	console.log(name); // Matt 
}
// var声明后, 整个函数内部都可以使用
function(){
	if (true) {
		let age = 26;
		console.log(age); // 26
	}
}
console.log(age); // ReferenceError: age 没有定义
//let声明的变量只存活与快内
```
## 条件声明  ***没听懂***
```js
在使用 var 声明变量时，由于声明会被提升，JavaScript 引擎会自动将多余的声明在作用域顶部合
并为一个声明 因为 let 的作用域是块，所以不可能检查前面是否已经使用 let 声明过同名变量，同

</script>

	let age = 36;
</script>
```
