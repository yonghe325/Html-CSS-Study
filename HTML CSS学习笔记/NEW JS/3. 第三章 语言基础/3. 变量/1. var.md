# var
+ 声明一个==函数作用域或全局作用域==的变量
	即申明之后, 在==整个函数内==都可以用
+ `var` 可以与 `function` 在同一作用域中声明同名变量
	`var` 声明的变量总是会覆盖函数声明的变量
+ `var` 不能与 `let const class import` 在==同一作用域==下声明同名变量
	因为 `var` 声明作用域不限于块，所以这也适用于以下情况：
	```js
	let a = 1;
	{
		  var a = 1; // SyntaxError: Identifier 'a' has already been declared
	}
	//异常, 因为var会在顶部声明, 导致重名
	//且let_a的作用域包含var_a的作用域
	//////////////////////////////////
	var a = 1;
	{
		let a = 2;
	}
	// 编译通过
	//let_a只在{}中存活,属于var_a的 子作用域
	//////////////////////////////////
	```
+ 函数体内var声明的变量, 可以与函数参数同名
	```js
	function foo(a) {
		  var a = 1;
		console.log(a);
	}
	foo(2); // 输出 1
	```
## 提升
使用`var`声明的变量, 无论出现在任何位置,
都会将声明提升到==函数作用域==的顶部
也就是在代码中的任何位置使用`var`声明变量都相当于==在顶部声明==它
```js
function foo() {
console.log(age);
var age = 26;
}
foo(); // undefined

//等价于

function foo() {
var age;
console.log(age);
age = 26;
}
foo(); // undefined
```
意味着变量可以在其声明之前被使用
+ 只有声明被提升, ==值不会被提升==
	所以只有运行到变量的赋值语句时, 该变量才会被初始化, 
	所以一个值var声明的变量可以被多次初始化, 但只会保留最后一个初始化
	```js
	function foo() {
		var age = 16;
		var age = 26;
		var age = 36;
		console.log(age);
	}
	foo(); // 36
	```
	在此之前 该变量的的值是 `undefined`
## for循环中的`var`
在`let`出现之前, 使用`var`声明的迭代变量会渗透到循环体外部
```js
for (var i = 0; i < 5; ++i) {
// 循环逻辑
}
console.log(i); // 5
```
使用`var`声明的迭代变量, 最常见的问题就是对==迭代变量的奇特声明和修改==
```js
for (var i = 0; i < 5; ++i) {
	setTimeout(() => console.log(i), 0)
}
// 不会输出0, 1, 2, 3, 4
// 实际上会输出 5, 5, 5, 5, 5
```
原因是setTimeout函数 内的函数 作为回调函数
会在for循环之后执行五次
而for循环结束之后, i始终等于 5 
至于setTimeout函数为什么在for循环之后执行
设计==JavaScript的事件循环机制==
