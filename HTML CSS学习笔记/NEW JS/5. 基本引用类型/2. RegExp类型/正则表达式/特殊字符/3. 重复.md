## ` * ` 匹配 ==零次== 到 多次
表示一个字符==可以完全不匹配== 或者 ==可以匹配多次==
## ` + ` 匹配 ==一次== 到 多次
至少匹配一次
## ` ? ` 表示该匹配可选
```RegExr
color, colour
/colou?r/g
```
表示`u`可有可无
## `{<number>}` 表示一个字符的出现次数
```
ber [beer] beeer beeeer
/be{2}r/g
//只匹配e出现两次的字符串
```
## `{<number>,}`  表示一个字符的==至少==出现次数
在数字后面加上`,`, 表示该字符至少出现几次
```
ber beer [beeer] [beeeer]
/be{3,}r/g
```
## `{<number>,<number>}` 表示一个字符出现次数的范围
```
[ber] [beer] [beeer] beeeer
/be{1,3}r/g
```
## 贪婪匹配
正则表达式==默认执行贪婪匹配==
这意味着匹配内容会尽可能长
## ` (*|+|{})? `懒惰匹配
与贪婪匹配不同
懒惰匹配在第一次匹配时停止
应用于`*|+|{}`
表示该重复尽量少匹配
```js
ber beer beeer beeeer
/.*?r/
//只会匹配ber
```
