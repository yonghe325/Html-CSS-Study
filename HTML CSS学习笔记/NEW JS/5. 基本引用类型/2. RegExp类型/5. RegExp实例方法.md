# exec() 捕获组
传入要检验的字符串作为`exec()`方法参数
`exec()`方法将会根据实例对应的正则表达式对该字符串进行匹配
若找到匹配项, ==则返回包含第一个匹配项的数组==
若未找到匹配项, 则返回`null`
## `exec`捕获数组
返回的数组虽然是 Array 的实例
但包含两个额外的属性：==index 和 input==
==index 是字符串中匹配模式的起始位置==
==input 是要查找的字符串==
这个数组的第一个元素是匹配整个模式的字符串，
其他元素是与表达式中的捕获组匹配的字符串
若没有捕获组, 则只有第一个元素
## index 捕获位置
`lastIndex`属性表示下一次开始查找的位置
### 全局标记 g
#### 匹配模式
全局模式下的 匹配策略 是==搜索模式==
==从 lastIndex 开始搜索, 当第一个字符开始没有匹配到时
会向后**跳过字符**, 直到匹配到或未匹配==
#### 返回
当RegExp实例==没有g==(global)标记时
每次对同一个字符串使用`exec()`方法
始终只会返回第一个匹配项
```js
let text = "cat, bat, sat, fat";
let pattern = /.at/;
//第一次调用
let matches = pattern.exec(text);
console.log(matches.index); // 0
console.log(matches[0]); // cat
console.log(pattern.lastIndex); // 0
//第二次调用
matches = pattern.exec(text);
console.log(matches.index); // 0
console.log(matches[0]); // cat
console.log(pattern.lastIndex); // 0

//始终会返回`cat`, 第一个匹配项
```
当RegExp实例==有g==(global)标记时
每次对同一个字符串使用`exec()`方法时
回依次向后匹配并返回
```js
let text = "cat, bat, sat, fat";
let pattern = /.at/g;
//第一次
let matches = pattern.exec(text);
console.log(matches.index); // 0
console.log(matches[0]); // cat
console.log(pattern.lastIndex); // 3
//第二次
matches = pattern.exec(text);
console.log(matches.index); // 5
console.log(matches[0]); // bat
console.log(pattern.lastIndex); // 8
//第三次
matches = pattern.exec(text);
console.log(matches.index); // 10
console.log(matches[0]); // sat
console.log(pattern.lastIndex); // 13
```
在全局匹配模式下
每次调用 `exec()`都会更新 `lastIndex` 值
以反映上次匹配的最后一个字符的索引
### 粘附标记 y
==粘附标记覆盖全局标记==
#### 匹配模式
如果模式设置了粘附标记 y
则每次调用 `exec()`就只会在 `lastIndex` 的位置上寻找匹配项
==且粘附模式是精确匹配, 不会跳过字符==
这意味着如果从`lastindex`位置开始, ==如果第一个字符未被匹配(空格), 则匹配失败==
且如果在粘附模式下匹配失败, 会将`lastIndex`设置为0
#### 返回
返回值同全局模式
```js
let text = "cat, bat, sat, fat";
let pattern = /.at/y;

let matches = pattern.exec(text);
console.log(matches.index); // 0
console.log(matches[0]); // cat
console.log(pattern.lastIndex); // 3
// 以索引 3 对应的字符开头找不到匹配项，因此 exec()返回 null
// exec()没找到匹配项，于是将 lastIndex 设置为 0
matches = pattern.exec(text);
console.log(matches); // null
console.log(pattern.lastIndex); // 0

// 向前设置 lastIndex 可以让粘附的模式通过 exec()找到下一个匹配项：
pattern.lastIndex = 5;
matches = pattern.exec(text);
console.log(matches.index); // 5
console.log(matches[0]); // bat
console.log(pattern.lastIndex); // 8
```
#### 手动实现跳过空格
```js
function tokenizeWithSkip(text) {
    const regex = /\w+/y;
    const tokens = [];
    let pos = 0;
    
    while (pos < text.length) {
        // 跳过空白字符
        if (/\s/.test(text[pos])) {
            pos++;
            //如果是空格
            //将lastindex位置后移
            continue;
        }
    
        // 设置起始位置
        regex.lastIndex = pos;
        const match = regex.exec(text);
        
        if (match && match.index === pos) {
            tokens.push(match[0]);
            pos = regex.lastIndex;
        } else {
            // 无法识别的字符
            throw new Error(`无法解析的字符: ${text[pos]} 在位置 ${pos}`);
        }
    }
    return tokens;
}

console.log(tokenizeWithSkip("abc 123 def"));  // ["abc", "123", "def"]
```
# `test()` 检验匹配
接收一个字符串
只检验其 是否匹配 `pattern`
如果输入的文本与模式匹配, 则参数返回 `true`, 否则返回 `false`
这个方法适用于只想测试模式是否匹配, 而不需要实际匹配内容的情况
```js
let text = "000-00-0000";
let pattern = /\d{3}-\d{2}-\d{4}/;

if (pattern.test(text)) {
	console.log("The pattern was matched.");
}
```
# 静态属性的替代方法

| 旧的 RegExp 属性              | 问题   | 现代替代方案               | 示例                                          |
| ------------------------- | ---- | -------------------- | ------------------------------------------- |
| **`RegExp.input`**        | 全局状态 | **匹配对象的 `input` 属性** | `match.input`                               |
| **`RegExp.lastMatch`**    | 全局状态 | **匹配对象的 `[0]`**      | `match[0]`                                  |
| **`RegExp.leftContext`**  | 全局状态 | **字符串切片**            | `text.slice(0, match.index)`                |
| **`RegExp.rightContext`** | 全局状态 | **字符串切片**            | `text.slice(match.index + match[0].length)` |
| **`RegExp.$1-$9`**        | 全局状态 | **匹配对象的 `[1]` 等**    | `match[1]`, `match[2]`                      |
| **`RegExp.lastParen`**    | 全局状态 | **解构赋值**             | `const [_, lastGroup] = match`              |
